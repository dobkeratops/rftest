  11  /*
  12   * This pretty-printer is a direct reimplementation of Philip Karlton's
  13   * Mesa pretty-printer, as described in appendix A of
  14   *
  15   *     STAN-CS-79-770: "Pretty Printing", by Derek C. Oppen.
  16   *     Stanford Department of Computer Science, 1979.
  17   *
  18   * The algorithm's aim is to break a stream into as few lines as possible
  19   * while respecting the indentation-consistency requirements of the enclosing
  20   * block, and avoiding breaking at silly places on block boundaries, for
  21   * example, between "x" and ")" in "x)".
  22   *
  23   * I am implementing this algorithm because it comes with 20 pages of
  24   * documentation explaining its theory, and because it addresses the set of
  25   * concerns I've seen other pretty-printers fall down on. Weirdly. Even though
  26   * it's 32 years old. What can I say?
  27   *
  28   * Despite some redundancies and quirks in the way it's implemented in that
  29   * paper, I've opted to keep the implementation here as similar as I can,
  30   * changing only what was blatantly wrong, a typo, or sufficiently
  31   * non-idiomatic rust that it really stuck out.
  32   *
  33   * In particular you'll see a certain amount of churn related to INTEGER vs.
  34   * CARDINAL in the Mesa implementation. Mesa apparently interconverts the two
  35   * somewhat readily? In any case, I've used uint for indices-in-buffers and
  36   * ints for character-sizes-and-indentation-offsets. This respects the need
  37   * for ints to "go negative" while carrying a pending-calculation balance, and
  38   * helps differentiate all the numbers flying around internally (slightly).
  39   *
  40   * I also inverted the indentation arithmetic used in the print stack, since
  41   * the Mesa implementation (somewhat randomly) stores the offset on the print
  42   * stack in terms of margin-col rather than col itself. I store col.
  43   *
  44   * I also implemented a small change in the String token, in that I store an
  45   * explicit length for the string. For most tokens this is just the length of
  46   * the accompanying string. But it's necessary to permit it to differ, for
  47   * encoding things that are supposed to "go on their own line" -- certain
  48   * classes of comment and blank-line -- where relying on adjacent
  49   * hardbreak-like Break tokens with long blankness indication doesn't actually
  50   * work. To see why, consider when there is a "thing that should be on its own
  51   * line" between two long blocks, say functions. If you put a hardbreak after
  52   * each function (or before each) and the breaking algorithm decides to break
  53   * there anyways (because the functions themselves are long) you wind up with
  54   * extra blank lines. If you don't put hardbreaks you can wind up with the
  55   * "thing which should be on its own line" not getting its own line in the
  56   * rare case of "really small functions" or such. This re-occurs with comments
  57   * and explicit blank lines. So in those cases we use a string with a payload
  58   * we want isolated to a line and an explicit length that's huge, surrounded
  59   * by two zero-length breaks. The algorithm will try its best to fit it on a
  60   * line (which it can't) and so naturally place the content on its own line to
  61   * avoid combining it with other lines and making matters even worse.
  62   */

 112      match t {

 114          Break(_) => return ~"BREAK",
 115          Begin(_) => return ~"BEGIN",

 117          Eof => return ~"EOF"

 121  pub fn buf_str(toks: Vec<Token> , szs: Vec<int> , left: uint, right: uint,
 122                 lim: uint) -> ~str {
 123      let n = toks.len();
 124      assert_eq!(n, szs.len());
 125      let mut i = left;
 126      let mut l = lim;
 127      let mut s = ~"[";
 128      while i != right && l != 0u {
 129          l -= 1u;
 130          if i != left {
 131              s.push_str(", ");
 132          }
 133          s.push_str(format!("{}={}", szs.get(i), tok_str(toks.get(i).clone())));
 134          i += 1u;
 135          i %= n;
 136      }
 137      s.push_char(']');
 138      return s;
 139  }

 182  /*
 183   * In case you do not have the paper, here is an explanation of what's going
 184   * on.
 185   *
 186   * There is a stream of input tokens flowing through this printer.
 187   *
 188   * The printer buffers up to 3N tokens inside itself, where N is linewidth.
 189   * Yes, linewidth is chars and tokens are multi-char, but in the worst
 190   * case every token worth buffering is 1 char long, so it's ok.
 191   *
 192   * Tokens are String, Break, and Begin/End to delimit blocks.
 193   *
 194   * Begin tokens can carry an offset, saying "how far to indent when you break
 195   * inside here", as well as a flag indicating "consistent" or "inconsistent"
 196   * breaking. Consistent breaking means that after the first break, no attempt
 197   * will be made to flow subsequent breaks together onto lines. Inconsistent
 198   * is the opposite. Inconsistent breaking example would be, say:
 199   *
 200   *  foo(hello, there, good, friends)
 201   *
 202   * breaking inconsistently to become
 203   *
 204   *  foo(hello, there
 205   *      good, friends);
 206   *
 207   * whereas a consistent breaking would yield:
 208   *
 209   *  foo(hello,
 210   *      there
 211   *      good,
 212   *      friends);
 213   *
 214   * That is, in the consistent-break blocks we value vertical alignment
 215   * more than the ability to cram stuff onto a line. But in all cases if it
 216   * can make a block a one-liner, it'll do so.
 217   *
 218   * Carrying on with high-level logic:
 219   *
 220   * The buffered tokens go through a ring-buffer, 'tokens'. The 'left' and
 221   * 'right' indices denote the active portion of the ring buffer as well as
 222   * describing hypothetical points-in-the-infinite-stream at most 3N tokens
 223   * apart (i.e. "not wrapped to ring-buffer boundaries"). The paper will switch
 224   * between using 'left' and 'right' terms to denote the wrapepd-to-ring-buffer
 225   * and point-in-infinite-stream senses freely.
 226   *
 227   * There is a parallel ring buffer, 'size', that holds the calculated size of
 228   * each token. Why calculated? Because for Begin/End pairs, the "size"
 229   * includes everything betwen the pair. That is, the "size" of Begin is
 230   * actually the sum of the sizes of everything between Begin and the paired
 231   * End that follows. Since that is arbitrarily far in the future, 'size' is
 232   * being rewritten regularly while the printer runs; in fact most of the
 233   * machinery is here to work out 'size' entries on the fly (and give up when
 234   * they're so obviously over-long that "infinity" is a good enough
 235   * approximation for purposes of line breaking).
 236   *
 237   * The "input side" of the printer is managed as an abstract process called
 238   * SCAN, which uses 'scan_stack', 'scan_stack_empty', 'top' and 'bottom', to
 239   * manage calculating 'size'. SCAN is, in other words, the process of
 240   * calculating 'size' entries.
 241   *
 242   * The "output side" of the printer is managed by an abstract process called
 243   * PRINT, which uses 'print_stack', 'margin' and 'space' to figure out what to
 244   * do with each token/size pair it consumes as it goes. It's trying to consume
 245   * the entire buffered window, but can't output anything until the size is >=
 246   * 0 (sizes are set to negative while they're pending calculation).
 247   *
 248   * So SCAN takes input and buffers tokens and pending calculations, while
 249   * PRINT gobbles up completed calculations and tokens from the buffer. The
 250   * theory is that the two can never get more than 3N tokens apart, because
 251   * once there's "obviously" too much data to fit on a line, in a size
 252   * calculation, SCAN will write "infinity" to the size and let PRINT consume
 253   * it.
 254   *
 255   * In this implementation (following the paper, again) the SCAN process is
 256   * the method called 'pretty_print', and the 'PRINT' process is the method
 257   * called 'print'.
 258   */
 259  pub struct Printer {
 260      pub out: ~io::Writer,
 261      buf_len: uint,
 262      margin: int, // width of lines we're constrained to
 263      space: int, // number of spaces left on line
 264      left: uint, // index of left side of input stream
 265      right: uint, // index of right side of input stream
 266      token: Vec<Token> , // ring-buffr stream goes through
 267      size: Vec<int> , // ring-buffer of calculated sizes
 268      left_total: int, // running size of stream "...left"
 269      right_total: int, // running size of stream "...right"
 270      // pseudo-stack, really a ring too. Holds the
 271      // primary-ring-buffers index of the Begin that started the
 272      // current block, possibly with the most recent Break after that
 273      // Begin (if there is any) on top of it. Stuff is flushed off the
 274      // bottom as it becomes irrelevant due to the primary ring-buffer
 275      // advancing.
 276      scan_stack: Vec<uint> ,
 277      scan_stack_empty: bool, // top==bottom disambiguator
 278      top: uint, // index of top of scan_stack
 279      bottom: uint, // index of bottom of scan_stack
 280      // stack of blocks-in-progress being flushed by print
 281      print_stack: Vec<PrintStackElem> ,
 282      // buffered indentation to avoid writing trailing whitespace
 283      pending_indentation: int,
 284  }

 287      pub fn last_token(&mut self) -> Token {
 288          (*self.token.get(self.right)).clone()
 289      }
 290      // be very careful with this!
 291      pub fn replace_last_token(&mut self, t: Token) {
 292          *self.token.get_mut(self.right) = t;
 293      }
 294      pub fn pretty_print(&mut self, t: Token) -> io::IoResult<()> {
 295          debug!("pp ~[{},{}]", self.left, self.right);
 296          match t {
 297            Eof => {
 298              if !self.scan_stack_empty {
 299                  self.check_stack(0);
 300                  let left = (*self.token.get(self.left)).clone();
 301                  let left_size = *self.size.get(self.left);
 302                  try!(self.advance_left(left, left_size));
 303              }
 304              self.indent(0);
 305              Ok(())
 306            }
 307            Begin(b) => {
 308              if self.scan_stack_empty {
 309                  self.left_total = 1;
 310                  self.right_total = 1;
 311                  self.left = 0u;
 312                  self.right = 0u;
 313              } else { self.advance_right(); }
 314              debug!("pp Begin({})/buffer ~[{},{}]",
 315                     b.offset, self.left, self.right);
 316              *self.token.get_mut(self.right) = t;
 317              *self.size.get_mut(self.right) = -self.right_total;
 318              self.scan_push(self.right);
 319              Ok(())
 320            }
 321            End => {
 322              if self.scan_stack_empty {
 323                  debug!("pp End/print ~[{},{}]", self.left, self.right);
 324                  self.print(t, 0)
 325              } else {
 326                  debug!("pp End/buffer ~[{},{}]", self.left, self.right);
 327                  self.advance_right();
 328                  *self.token.get_mut(self.right) = t;
 329                  *self.size.get_mut(self.right) = -1;
 330                  self.scan_push(self.right);
 331                  Ok(())
 332              }
 333            }
 334            Break(b) => {
 335              if self.scan_stack_empty {
 336                  self.left_total = 1;
 337                  self.right_total = 1;
 338                  self.left = 0u;
 339                  self.right = 0u;
 340              } else { self.advance_right(); }
 341              debug!("pp Break({})/buffer ~[{},{}]",
 342                     b.offset, self.left, self.right);
 343              self.check_stack(0);
 344              self.scan_push(self.right);
 345              *self.token.get_mut(self.right) = t;
 346              *self.size.get_mut(self.right) = -self.right_total;
 347              self.right_total += b.blank_space;
 348              Ok(())
 349            }
 350            String(ref s, len) => {
 351              if self.scan_stack_empty {
 352                  debug!("pp String('{}')/print ~[{},{}]",
 353                         *s, self.left, self.right);
 354                  self.print(t.clone(), len)
 355              } else {
 356                  debug!("pp String('{}')/buffer ~[{},{}]",
 357                         *s, self.left, self.right);
 358                  self.advance_right();
 359                  *self.token.get_mut(self.right) = t.clone();
 360                  *self.size.get_mut(self.right) = len;
 361                  self.right_total += len;
 362                  self.check_stream()
 363              }
 364            }
 365          }
 366      }
 367      pub fn check_stream(&mut self) -> io::IoResult<()> {
 368          debug!("check_stream ~[{}, {}] with left_total={}, right_total={}",
 369                 self.left, self.right, self.left_total, self.right_total);
 370          if self.right_total - self.left_total > self.space {
 371              debug!("scan window is {}, longer than space on line ({})",
 372                     self.right_total - self.left_total, self.space);
 373              if !self.scan_stack_empty {
 374                  if self.left == *self.scan_stack.get(self.bottom) {
 375                      debug!("setting {} to infinity and popping", self.left);
 376                      let scanned = self.scan_pop_bottom();
 377                      *self.size.get_mut(scanned) = SIZE_INFINITY;
 378                  }
 379              }
 380              let left = (*self.token.get(self.left)).clone();
 381              let left_size = *self.size.get(self.left);
 382              try!(self.advance_left(left, left_size));
 383              if self.left != self.right {
 384                  try!(self.check_stream());
 385              }
 386          }
 387          Ok(())
 388      }
 389      pub fn scan_push(&mut self, x: uint) {
 390          debug!("scan_push {}", x);
 391          if self.scan_stack_empty {
 392              self.scan_stack_empty = false;
