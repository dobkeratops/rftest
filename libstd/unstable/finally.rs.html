  11  /*!
  12  The Finally trait provides a method, `finally` on
  13  stack closures that emulates Java-style try/finally blocks.
  14  
  15  Using the `finally` method is sometimes convenient, but the type rules
  16  prohibit any shared, mutable state between the "try" case and the
  17  "finally" case. For advanced cases, the `try_finally` function can
  18  also be used. See that function for more details.
  19  
  20  # Example
  21  
  22  ```
  23  use std::unstable::finally::Finally;
  24  # fn always_run_this() {}
  25  
  26  (|| {
  27      // ...
  28  }).finally(|| {
  29      always_run_this();
  30  })

  58  /**
  59   * The most general form of the `finally` functions. The function
  60   * `try_fn` will be invoked first; whether or not it fails, the
  61   * function `finally_fn` will be invoked next. The two parameters
  62   * `mutate` and `drop` are used to thread state through the two
  63   * closures. `mutate` is used for any shared, mutable state that both
  64   * closures require access to; `drop` is used for any state that the
  65   * `try_fn` requires ownership of.
  66   *
  67   * **WARNING:** While shared, mutable state between the try and finally
  68   * function is often necessary, one must be very careful; the `try`
  69   * function could have failed at any point, so the values of the shared
  70   * state may be inconsistent.
  71   *
  72   * # Example
  73   *
  74   * ```
  75   * use std::unstable::finally::try_finally;
  76   *
  77   * struct State<'a> { buffer: &'a mut [u8], len: uint }
  78   * # let mut buf = [];
  79   * let mut state = State { buffer: buf, len: 0 };

  81   *     &mut state, (),

  85   *     |state| {
