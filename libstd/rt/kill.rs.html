  11 /*!
  12 
  13 Task death: asynchronous killing, linked failure, exit code propagation.
  14 
  15 This file implements two orthogonal building-blocks for communicating failure
  16 between tasks. One is 'linked failure' or 'task killing', that is, a failing
  17 task causing other tasks to fail promptly (even those that are blocked on
  18 pipes or I/O). The other is 'exit code propagation', which affects the result
  19 observed by the parent of a task::try task that itself spawns child tasks
  20 (such as any #[test] function). In both cases the data structures live in
  21 KillHandle.
  22 
  23 I. Task killing.
  24 
  25 The model for killing involves two atomic flags, the "kill flag" and the
  26 "unkillable flag". Operations on the kill flag include:
  27 
  28 - In the taskgroup code (task/spawn.rs), tasks store a clone of their
  29   KillHandle in their shared taskgroup. Another task in the group that fails
  30   will use that handle to call kill().
  31 - When a task blocks, it turns its ~Task into a BlockedTask by storing a
  32   the transmuted ~Task pointer inside the KillHandle's kill flag. A task
  33   trying to block and a task trying to kill it can simultaneously access the
  34   kill flag, after which the task will get scheduled and fail (no matter who
  35   wins the race). Likewise, a task trying to wake a blocked task normally and
  36   a task trying to kill it can simultaneously access the flag; only one will
  37   get the task to reschedule it.
  38 
  39 Operations on the unkillable flag include:
  40 
  41 - When a task becomes unkillable, it swaps on the flag to forbid any killer
  42   from waking it up while it's blocked inside the unkillable section. If a
  43   kill was already pending, the task fails instead of becoming unkillable.
  44 - When a task is done being unkillable, it restores the flag to the normal
  45   running state. If a kill was received-but-blocked during the unkillable
  46   section, the task fails at this later point.
  47 - When a task tries to kill another task, before swapping on the kill flag, it
  48   first swaps on the unkillable flag, to see if it's "allowed" to wake up the
  49   task. If it isn't, the killed task will receive the signal when it becomes
  50   killable again. (Of course, a task trying to wake the task normally (e.g.
  51   sending on a channel) does not access the unkillable flag at all.)
  52 
  53 Why do we not need acquire/release barriers on any of the kill flag swaps?
  54 This is because barriers establish orderings between accesses on different
  55 memory locations, but each kill-related operation is only a swap on a single
  56 location, so atomicity is all that matters. The exception is kill(), which
  57 does a swap on both flags in sequence. kill() needs no barriers because it
  58 does not matter if its two accesses are seen reordered on another CPU: if a
  59 killer does perform both writes, it means it saw a KILL_RUNNING in the
  60 unkillable flag, which means an unkillable task will see KILL_KILLED and fail
  61 immediately (rendering the subsequent write to the kill flag unnecessary).
  62 
  63 II. Exit code propagation.
  64 
  65 FIXME(#7544): Decide on the ultimate model for this and document it.
  66 
  67 */

  91 /// A handle to a blocked task. Usually this means having the ~Task pointer by
  92 /// ownership, but if the task is killable, a killer can steal it at any time.
  93 pub enum BlockedTask {
  94     Unkillable(~Task),
  95     Killable(KillFlagHandle),
  96 }

  98 // FIXME(#7544)(bblum): think about the cache efficiency of this
  99 struct KillHandleInner {
 100     // Is the task running, blocked, or killed? Possible values:
 101     // * KILL_RUNNING    - Not unkillable, no kill pending.
 102     // * KILL_KILLED     - Kill pending.
 103     // * <ptr>           - A transmuted blocked ~Task pointer.
 104     // This flag is refcounted because it may also be referenced by a blocking
 105     // concurrency primitive, used to wake the task normally, whose reference
 106     // may outlive the handle's if the task is killed.
 107     killed: KillFlagHandle,
 108     // Has the task deferred kill signals? This flag guards the above one.
 109     // Possible values:
 110     // * KILL_RUNNING    - Not unkillable, no kill pending.
 111     // * KILL_KILLED     - Kill pending.
 112     // * KILL_UNKILLABLE - Kill signals deferred.
 113     unkillable: AtomicUint,
 114 
 115     // Shared state between task and children for exit code propagation. These
 116     // are here so we can re-use the kill handle to implement watched children
 117     // tasks. Using a separate Arc-like would introduce extra atomic adds/subs
 118     // into common spawn paths, so this is just for speed.
 119 
 120     // Locklessly accessed; protected by the enclosing refcount's barriers.
 121     any_child_failed: bool,
 122     // A lazy list, consuming which may unwrap() many child tombstones.
 123     child_tombstones: Option<~fn() -> bool>,
 124     // Protects multiple children simultaneously creating tombstones.
 125     graveyard_lock: LittleLock,
 126 }

 132 /// Per-task state related to task death, killing, failure, etc.
 133 pub struct Death {
 134     // Shared among this task, its watched children, and any linked tasks who
 135     // might kill it. This is optional so we can take it by-value at exit time.
 136     kill_handle:     Option<KillHandle>,
 137     // Handle to a watching parent, if we have one, for exit code propagation.
 138     watching_parent: Option<KillHandle>,
 139     // Action to be done with the exit code. If set, also makes the task wait
 140     // until all its watched children exit before collecting the status.
 141     on_exit:         Option<~fn(bool)>,
 142     // nesting level counter for task::unkillable calls (0 == killable).
 143     unkillable:      int,
 144     // nesting level counter for unstable::atomically calls (0 == can yield).
 145     wont_sleep:      int,
 146     // A "spare" handle to the kill flag inside the kill handle. Used during
 147     // blocking/waking as an optimization to avoid two xadds on the refcount.
 148     spare_kill_flag: Option<KillFlagHandle>,
 149 }

 151 impl Drop for KillFlag {
 152     // Letting a KillFlag with a task inside get dropped would leak the task.
 153     // We could free it here, but the task should get awoken by hand somehow.
 154     fn drop(&self) {
 155         match self.load(Relaxed) {
 156             KILL_RUNNING | KILL_KILLED => { },
 157             _ => rtabort!("can't drop kill flag with a blocked task inside!"),
 158         }
 159     }
 160 }
 161 
 162 // Whenever a task blocks, it swaps out its spare kill flag to use as the
 163 // blocked task handle. So unblocking a task must restore that spare.
 164 unsafe fn revive_task_ptr(task_ptr: uint, spare_flag: Option<KillFlagHandle>) -> ~Task {
 165     let mut task: ~Task = cast::transmute(task_ptr);
 166     if task.death.spare_kill_flag.is_none() {
 167         task.death.spare_kill_flag = spare_flag;
 168     } else {
 169         // A task's spare kill flag is not used for blocking in one case:
 170         // when an unkillable task blocks on select. In this case, a separate
 171         // one was created, which we now discard.
 172         rtassert!(task.death.unkillable > 0);
 173     }
 174     task
 175 }

 177 impl BlockedTask {
 178     /// Returns Some if the task was successfully woken; None if already killed.
 179     pub fn wake(self) -> Option<~Task> {
 180         match self {
 181             Unkillable(task) => Some(task),
 182             Killable(flag_arc) => {
 183                 let flag = unsafe { &mut **flag_arc.get() };
 184                 match flag.swap(KILL_RUNNING, Relaxed) {
 185                     KILL_RUNNING => None, // woken from select(), perhaps
 186                     KILL_KILLED  => None, // a killer stole it already
 187                     task_ptr     =>
 188                         Some(unsafe { revive_task_ptr(task_ptr, Some(flag_arc)) })
 189                 }
 190             }
 191         }
 192     }

 194     /// Create a blocked task, unless the task was already killed.
 195     pub fn try_block(mut task: ~Task) -> Either<~Task, BlockedTask> {
 196         if task.death.unkillable > 0 {
 197             Right(Unkillable(task))
 198         } else {
 199             rtassert!(task.death.kill_handle.is_some());
 200             unsafe {
 201                 // The inverse of 'revive', above, occurs here.
 202                 // The spare kill flag will usually be Some, unless the task was
 203                 // already killed, in which case the killer will have deferred
 204                 // creating a new one until whenever it blocks during unwinding.
 205                 let flag_arc = match task.death.spare_kill_flag.take() {
 206                     Some(spare_flag) => spare_flag,
 207                     None => {
 208                         // FIXME(#7544): Uncomment this when terminate_current_task
 209                         // stops being *terrible*. That's the only place that violates
 210                         // the assumption of "becoming unkillable will fail if the
 211                         // task was killed".
 212                         // rtassert!(task.unwinder.unwinding);
 213                         (*task.death.kill_handle.get_ref().get()).killed.clone()
 214                     }
 215                 };
 216                 let flag     = &mut **flag_arc.get();
 217                 let task_ptr = cast::transmute(task);
 218                 // Expect flag to contain RUNNING. If KILLED, it should stay KILLED.
 219                 match flag.compare_and_swap(KILL_RUNNING, task_ptr, Relaxed) {
 220                     KILL_RUNNING => Right(Killable(flag_arc)),
 221                     KILL_KILLED  => Left(revive_task_ptr(task_ptr, Some(flag_arc))),
 222                     x            => rtabort!("can't block task! kill flag = %?", x),
 223                 }
 224             }
 225         }
 226     }

 228     /// Converts one blocked task handle to a list of many handles to the same.
 229     pub fn make_selectable(self, num_handles: uint) -> ~[BlockedTask] {
 230         let handles = match self {
 231             Unkillable(task) => {
 232                 let flag = unsafe { KillFlag(AtomicUint::new(cast::transmute(task))) };
 233                 UnsafeAtomicRcBox::newN(flag, num_handles)
 234             }
 235             Killable(flag_arc) => flag_arc.cloneN(num_handles),
 236         };
 237         // Even if the task was unkillable before, we use 'Killable' because
 238         // multiple pipes will have handles. It does not really mean killable.
 239         handles.consume_iter().transform(|x| Killable(x)).collect()
 240     }

 249     /// Convert to an unsafe uint value. Useful for storing in a pipe's state flag.
 250     #[inline]
 251     pub unsafe fn cast_to_uint(self) -> uint {
 252         // Use the low bit to distinguish the enum variants, to save a second
 253         // allocation in the indestructible case.
 254         match self {
 255             Unkillable(task) => {
 256                 let blocked_task_ptr: uint = cast::transmute(task);
 257                 rtassert!(blocked_task_ptr & 0x1 == 0);
 258                 blocked_task_ptr
 259             },
 260             Killable(flag_arc) => {
 261                 let blocked_task_ptr: uint = cast::transmute(~flag_arc);
 262                 rtassert!(blocked_task_ptr & 0x1 == 0);
 263                 blocked_task_ptr | 0x1
 264             }
 265         }
 266     }

 268     /// Convert from an unsafe uint value. Useful for retrieving a pipe's state flag.
 269     #[inline]
 270     pub unsafe fn cast_from_uint(blocked_task_ptr: uint) -> BlockedTask {
 271         if blocked_task_ptr & 0x1 == 0 {
 272             Unkillable(cast::transmute(blocked_task_ptr))
 273         } else {
 274             let ptr: ~KillFlagHandle = cast::transmute(blocked_task_ptr & !0x1);
 275             match ptr {
 276                 ~flag_arc => Killable(flag_arc)
 277             }
 278         }
 279     }
 280 }

 293 impl KillHandle {
 294     pub fn new() -> (KillHandle, KillFlagHandle) {
 295         let (flag, flag_clone) =
 296             UnsafeAtomicRcBox::new2(KillFlag(AtomicUint::new(KILL_RUNNING)));
 297         let handle = KillHandle(UnsafeAtomicRcBox::new(KillHandleInner {
 298             // Linked failure fields
 299             killed:     flag,
 300             unkillable: AtomicUint::new(KILL_RUNNING),
 301             // Exit code propagation fields
 302             any_child_failed: false,
 303             child_tombstones: None,
 304             graveyard_lock:   LittleLock::new(),
 305         }));
 306         (handle, flag_clone)
 307     }

 309     // Will begin unwinding if a kill signal was received, unless already_failing.
 310     // This can't be used recursively, because a task which sees a KILLED
 311     // signal must fail immediately, which an already-unkillable task can't do.
 312     #[inline]
 313     pub fn inhibit_kill(&mut self, already_failing: bool) {
 314         let inner = unsafe { &mut *self.get() };
 315         // Expect flag to contain RUNNING. If KILLED, it should stay KILLED.
 316         // FIXME(#7544)(bblum): is it really necessary to prohibit double kill?
 317         match inner.unkillable.compare_and_swap(KILL_RUNNING, KILL_UNKILLABLE, Relaxed) {
 318             KILL_RUNNING    => { }, // normal case
 319             KILL_KILLED     => if !already_failing { fail!(KILLED_MSG) },
 320             _               => rtabort!("inhibit_kill: task already unkillable"),
 321         }
 322     }
 323 
 324     // Will begin unwinding if a kill signal was received, unless already_failing.
 325     #[inline]
 326     pub fn allow_kill(&mut self, already_failing: bool) {
 327         let inner = unsafe { &mut *self.get() };
 328         // Expect flag to contain UNKILLABLE. If KILLED, it should stay KILLED.
 329         // FIXME(#7544)(bblum): is it really necessary to prohibit double kill?
 330         match inner.unkillable.compare_and_swap(KILL_UNKILLABLE, KILL_RUNNING, Relaxed) {
 331             KILL_UNKILLABLE => { }, // normal case
 332             KILL_KILLED     => if !already_failing { fail!(KILLED_MSG) },
 333             _               => rtabort!("allow_kill: task already killable"),
 334         }
 335     }
 336 
 337     // Send a kill signal to the handle's owning task. Returns the task itself
 338     // if it was blocked and needs punted awake. To be called by other tasks.
 339     pub fn kill(&mut self) -> Option<~Task> {
 340         let inner = unsafe { &mut *self.get() };
 341         if inner.unkillable.swap(KILL_KILLED, Relaxed) == KILL_RUNNING {
 342             // Got in. Allowed to try to punt the task awake.
 343             let flag = unsafe { &mut *inner.killed.get() };
 344             match flag.swap(KILL_KILLED, Relaxed) {
 345                 // Task either not blocked or already taken care of.
 346                 KILL_RUNNING | KILL_KILLED => None,
 347                 // Got ownership of the blocked task.
 348                 // While the usual 'wake' path can just pass back the flag
 349                 // handle, we (the slower kill path) haven't an extra one lying
 350                 // around. The task will wake up without a spare.
 351                 task_ptr => Some(unsafe { revive_task_ptr(task_ptr, None) }),
 352             }
 353         } else {
 354             // Otherwise it was either unkillable or already killed. Somebody
 355             // else was here first who will deal with the kill signal.
 356             None
 357         }
 358     }

 360     #[inline]
 361     pub fn killed(&self) -> bool {
 362         // Called every context switch, so shouldn't report true if the task
 363         // is unkillable with a kill signal pending.
 364         let inner = unsafe { &*self.get() };
 365         let flag  = unsafe { &*inner.killed.get() };
 366         // A barrier-related concern here is that a task that gets killed
 367         // awake needs to see the killer's write of KILLED to this flag. This
 368         // is analogous to receiving a pipe payload; the appropriate barrier
 369         // should happen when enqueueing the task.
 370         flag.load(Relaxed) == KILL_KILLED
 371     }

 373     pub fn notify_immediate_failure(&mut self) {
 374         // A benign data race may happen here if there are failing sibling
 375         // tasks that were also spawned-watched. The refcount's write barriers
 376         // in UnsafeAtomicRcBox ensure that this write will be seen by the
 377         // unwrapper/destructor, whichever task may unwrap it.
 378         unsafe { (*self.get()).any_child_failed = true; }
 379     }
 380 
 381     // For use when a task does not need to collect its children's exit
 382     // statuses, but the task has a parent which might want them.
 383     pub fn reparent_children_to(self, parent: &mut KillHandle) {
 384         // Optimistic path: If another child of the parent's already failed,
 385         // we don't need to worry about any of this.
 386         if unsafe { (*parent.get()).any_child_failed } {
 387             return;
 388         }
 389 
 390         // Try to see if all our children are gone already.
 391         match self.try_unwrap() {
 392             // Couldn't unwrap; children still alive. Reparent entire handle as
 393             // our own tombstone, to be unwrapped later.
 394             Left(this) => {
 395                 let this = Cell::new(this); // :(
 396                 do add_lazy_tombstone(parent) |other_tombstones| {
 397                     let this = Cell::new(this.take()); // :(
 398                     let others = Cell::new(other_tombstones); // :(
 399                     || {
 400                         // Prefer to check tombstones that were there first,
 401                         // being "more fair" at the expense of tail-recursion.
 402                         others.take().map_consume_default(true, |f| f()) && {
 403                             let mut inner = this.take().unwrap();
 404                             (!inner.any_child_failed) &&
 405                                 inner.child_tombstones.take_map_default(true, |f| f())
 406                         }
 407                     }
 408                 }
 409             }
 410             // Whether or not all children exited, one or more already failed.
 411             Right(KillHandleInner { any_child_failed: true, _ }) => {
 412                 parent.notify_immediate_failure();
 413             }
 414             // All children exited, but some left behind tombstones that we
 415             // don't want to wait on now. Give them to our parent.
 416             Right(KillHandleInner { any_child_failed: false,
 417                                     child_tombstones: Some(f), _ }) => {
 418                 let f = Cell::new(f); // :(
 419                 do add_lazy_tombstone(parent) |other_tombstones| {
 420                     let f = Cell::new(f.take()); // :(
 421                     let others = Cell::new(other_tombstones); // :(
 422                     || {
 423                         // Prefer fairness to tail-recursion, as in above case.
 424                         others.take().map_consume_default(true, |f| f()) &&
 425                             f.take()()
 426                     }
 427                 }
 428             }
 429             // All children exited, none failed. Nothing to do!
 430             Right(KillHandleInner { any_child_failed: false,
 431                                     child_tombstones: None, _ }) => { }
 432         }
 433 
 434         // NB: Takes a pthread mutex -- 'blk' not allowed to reschedule.
 435         #[inline]
 436         fn add_lazy_tombstone(parent: &mut KillHandle,
 437                               blk: &fn(Option<~fn() -> bool>) -> ~fn() -> bool) {
 438 
 439             let inner: &mut KillHandleInner = unsafe { &mut *parent.get() };
 440             unsafe {
 441                 do inner.graveyard_lock.lock {
 442                     // Update the current "head node" of the lazy list.
 443                     inner.child_tombstones =
 444                         Some(blk(util::replace(&mut inner.child_tombstones, None)));
 445                 }
 446             }
 447         }
 448     }
 449 }

 477     /// Collect failure exit codes from children and propagate them to a parent.
 478     pub fn collect_failure(&mut self, mut success: bool, group: Option<Taskgroup>) {
 479         // This may run after the task has already failed, so even though the
 480         // task appears to need to be killed, the scheduler should not fail us
 481         // when we block to unwrap.
 482         // (XXX: Another less-elegant reason for doing this is so that the use
 483         // of the LittleLock in reparent_children_to doesn't need to access the
 484         // unkillable flag in the kill_handle, since we'll have removed it.)
 485         rtassert!(self.unkillable == 0);
 486         self.unkillable = 1;
 487 
 488         // FIXME(#7544): See corresponding fixme at the callsite in task.rs.
 489         // NB(#8192): Doesn't work with "let _ = ..."
 490         { use util; util::ignore(group); }
 491 
 492         // Step 1. Decide if we need to collect child failures synchronously.
 493         do self.on_exit.take_map |on_exit| {
 494             if success {
 495                 // We succeeded, but our children might not. Need to wait for them.
 496                 let mut inner = self.kill_handle.take_unwrap().unwrap();
 497                 if inner.any_child_failed {
 498                     success = false;
 499                 } else {
 500                     // Lockless access to tombstones protected by unwrap barrier.
 501                     success = inner.child_tombstones.take_map_default(true, |f| f());
 502                 }
 503             }
 504             on_exit(success);
 505         };
 506 
 507         // Step 2. Possibly alert possibly-watching parent to failure status.
 508         // Note that as soon as parent_handle goes out of scope, the parent
 509         // can successfully unwrap its handle and collect our reported status.
 510         do self.watching_parent.take_map |mut parent_handle| {
 511             if success {
 512                 // Our handle might be None if we had an exit callback, and
 513                 // already unwrapped it. But 'success' being true means no
 514                 // child failed, so there's nothing to do (see below case).
 515                 do self.kill_handle.take_map |own_handle| {
 516                     own_handle.reparent_children_to(&mut parent_handle);
 517                 };
 518             } else {
 519                 // Can inform watching parent immediately that we failed.
 520                 // (Note the importance of non-failing tasks NOT writing
 521                 // 'false', which could obscure another task's failure.)
 522                 parent_handle.notify_immediate_failure();
 523             }
 524         };
 525 
 526         // Can't use allow_kill directly; that would require the kill handle.
 527         rtassert!(self.unkillable == 1);
 528         self.unkillable = 0;
 529     }

 531     /// Fails if a kill signal was received.
 532     #[inline]
 533     pub fn check_killed(&self, already_failing: bool) {
 534         match self.kill_handle {
 535             Some(ref kill_handle) =>
 536                 // The task may be both unkillable and killed if it does some
 537                 // synchronization during unwinding or cleanup (for example,
 538                 // sending on a notify port). In that case failing won't help.
 539                 if self.unkillable == 0 && (!already_failing) && kill_handle.killed() {
 540                     fail!(KILLED_MSG);
 541                 },
 542             // This may happen during task death (see comments in collect_failure).
 543             None => rtassert!(self.unkillable > 0),
 544         }
 545     }
