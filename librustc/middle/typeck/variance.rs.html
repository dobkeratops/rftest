   11  /*!
   12  
   13  This file infers the variance of type and lifetime parameters. The
   14  algorithm is taken from Section 4 of the paper "Taming the Wildcards:
   15  Combining Definition- and Use-Site Variance" published in PLDI'11 and
   16  written by Altidor et al., and hereafter referred to as The Paper.
   17  
   18  This inference is explicitly designed *not* to consider the uses of
   19  types within code. To determine the variance of type parameters
   20  defined on type `X`, we only consider the definition of the type `X`
   21  and the definitions of any types it references.
   22  
   23  We only infer variance for type parameters found on *types*: structs,
   24  enums, and traits. We do not infer variance for type parameters found
   25  on fns or impls. This is because those things are not type definitions
   26  and variance doesn't really make sense in that context.
   27  
   28  It is worth covering what variance means in each case. For structs and
   29  enums, I think it is fairly straightforward. The variance of the type
   30  or lifetime parameters defines whether `T<A>` is a subtype of `T<B>`
   31  (resp. `T<'a>` and `T<'b>`) based on the relationship of `A` and `B`
   32  (resp. `'a` and `'b`). (FIXME #3598 -- we do not currently make use of
   33  the variances we compute for type parameters.)
   34  
   35  ### Variance on traits
   36  
   37  The meaning of variance for trait parameters is more subtle and worth
   38  expanding upon. There are in fact two uses of the variance values we
   39  compute.
   40  
   41  #### Trait variance and object types
   42  
   43  The first is for object types. Just as with structs and enums, we can
   44  decide the subtyping relationship between two object types `&Trait<A>`
   45  and `&Trait<B>` based on the relationship of `A` and `B`. Note that
   46  for object types we ignore the `Self` type parameter -- it is unknown,
   47  and the nature of dynamic dispatch ensures that we will always call a
   48  function that is expected the appropriate `Self` type. However, we
   49  must be careful with the other type parameters, or else we could end
   50  up calling a function that is expecting one type but provided another.
   51  
   52  To see what I mean, consider a trait like so:
   53  
   54      trait ConvertTo<A> {
   55          fn convertTo(&self) -> A;
   56      }
   57  
   58  Intuitively, If we had one object `O=&ConvertTo<Object>` and another
   59  `S=&ConvertTo<String>`, then `S <: O` because `String <: Object`
   60  (presuming Java-like "string" and "object" types, my go to examples
   61  for subtyping). The actual algorithm would be to compare the
   62  (explicit) type parameters pairwise respecting their variance: here,
   63  the type parameter A is covariant (it appears only in a return
   64  position), and hence we require that `String <: Object`.
   65  
   66  You'll note though that we did not consider the binding for the
   67  (implicit) `Self` type parameter: in fact, it is unknown, so that's
   68  good. The reason we can ignore that parameter is precisely because we
   69  don't need to know its value until a call occurs, and at that time (as
   70  you said) the dynamic nature of virtual dispatch means the code we run
   71  will be correct for whatever value `Self` happens to be bound to for
   72  the particular object whose method we called. `Self` is thus different
   73  from `A`, because the caller requires that `A` be known in order to
   74  know the return type of the method `convertTo()`. (As an aside, we
   75  have rules preventing methods where `Self` appears outside of the
   76  receiver position from being called via an object.)
   77  
   78  #### Trait variance and vtable resolution
   79  
   80  But traits aren't only used with objects. They're also used when
   81  deciding whether a given impl satisfies a given trait bound (or should
   82  be -- FIXME #5781). To set the scene here, imagine I had a function:
   83  
   84      fn convertAll<A,T:ConvertTo<A>>(v: &[T]) {
   85          ...
   86      }
   87  
   88  Now imagine that I have an implementation of `ConvertTo` for `Object`:
   89  
   90      impl ConvertTo<int> for Object { ... }
   91  
   92  And I want to call `convertAll` on an array of strings. Suppose
   93  further that for whatever reason I specifically supply the value of
   94  `String` for the type parameter `T`:
   95  
   96      let mut vector = ~["string", ...];
   97      convertAll::<int, String>(v);
   98  
   99  Is this legal? To put another way, can we apply the `impl` for
  100  `Object` to the type `String`? The answer is yes, but to see why
  101  we have to expand out what will happen:
  102  
  103  - `convertAll` will create a pointer to one of the entries in the
  104    vector, which will have type `&String`
  105  - It will then call the impl of `convertTo()` that is intended
  106    for use with objects. This has the type:
  107  
  108        fn(self: &Object) -> int
  109  
  110    It is ok to provide a value for `self` of type `&String` because
  111    `&String <: &Object`.
  112  
  113  OK, so intuitively we want this to be legal, so let's bring this back
  114  to variance and see whether we are computing the correct result. We
  115  must first figure out how to phrase the question "is an impl for
  116  `Object,int` usable where an impl for `String,int` is expected?"
  117  
  118  Maybe it's helpful to think of a dictionary-passing implementation of
  119  type classes. In that case, `convertAll()` takes an implicit parameter
  120  representing the impl. In short, we *have* an impl of type:
  121  
  122      V_O = ConvertTo<int> for Object
  123  
  124  and the function prototype expects an impl of type:
  125  
  126      V_S = ConvertTo<int> for String
  127  
  128  As with any argument, this is legal if the type of the value given
  129  (`V_O`) is a subtype of the type expected (`V_S`). So is `V_O <: V_S`?
  130  The answer will depend on the variance of the various parameters. In
  131  this case, because the `Self` parameter is contravariant and `A` is
  132  covariant, it means that:
  133  
  134      V_O <: V_S iff
  135          int <: int
  136          String <: Object
  137  
  138  These conditions are satisfied and so we are happy.
  139  
  140  ### The algorithm
  141  
  142  The basic idea is quite straightforward. We iterate over the types
  143  defined and, for each use of a type parameter X, accumulate a
  144  constraint indicating that the variance of X must be valid for the
  145  variance of that use site. We then iteratively refine the variance of
  146  X until all constraints are met. There is *always* a sol'n, because at
  147  the limit we can declare all type parameters to be invariant and all
  148  constraints will be satisfied.
  149  
  150  As a simple example, consider:
  151  
  152      enum Option<A> { Some(A), None }
  153      enum OptionalFn<B> { Some(|B|), None }
  154      enum OptionalMap<C> { Some(|C| -> C), None }
  155  
  156  Here, we will generate the constraints:
  157  
  158      1. V(A) <= +
  159      2. V(B) <= -
  160      3. V(C) <= +
  161      4. V(C) <= -
  162  
  163  These indicate that (1) the variance of A must be at most covariant;
  164  (2) the variance of B must be at most contravariant; and (3, 4) the
  165  variance of C must be at most covariant *and* contravariant. All of these
  166  results are based on a variance lattice defined as follows:
  167  
  168        *      Top (bivariant)
  169     -     +
  170        o      Bottom (invariant)
  171  
  172  Based on this lattice, the solution V(A)=+, V(B)=-, V(C)=o is the
  173  optimal solution. Note that there is always a naive solution which
  174  just declares all variables to be invariant.
  175  
  176  You may be wondering why fixed-point iteration is required. The reason
  177  is that the variance of a use site may itself be a function of the
  178  variance of other type parameters. In full generality, our constraints
  179  take the form:
  180  
  181      V(X) <= Term
  182      Term := + | - | * | o | V(X) | Term x Term
  183  
  184  Here the notation V(X) indicates the variance of a type/region
  185  parameter `X` with respect to its defining class. `Term x Term`
  186  represents the "variance transform" as defined in the paper:
  187  
  188    If the variance of a type variable `X` in type expression `E` is `V2`
  189    and the definition-site variance of the [corresponding] type parameter
  190    of a class `C` is `V1`, then the variance of `X` in the type expression
  191    `C<E>` is `V3 = V1.xform(V2)`.
  192  
  193  */

  207  pub fn infer_variance(tcx: &ty::ctxt,
  208                        krate: &ast::Crate) {
  209      let mut arena = arena::Arena::new();
  210      let terms_cx = determine_parameters_to_be_inferred(tcx, &mut arena, krate);
  211      let constraints_cx = add_constraints_from_crate(terms_cx, krate);
  212      solve_constraints(constraints_cx);
  213  }

  215  /**************************************************************************
  216   * Representing terms
  217   *
  218   * Terms are structured as a straightforward tree. Rather than rely on
  219   * GC, we allocate terms out of a bounded arena (the lifetime of this
  220   * arena is the lifetime 'a that is threaded around).
  221   *
  222   * We assign a unique index to each type/region parameter whose variance
  223   * is to be inferred. We refer to such variables as "inferreds". An
  224   * `InferredIndex` is a newtype'd int representing the index of such
  225   * a variable.
  226   */
  227  
  228  type VarianceTermPtr<'a> = &'a VarianceTerm<'a>;
  229  
  230  struct InferredIndex(uint);
  231  
  232  enum VarianceTerm<'a> {
  233      ConstantTerm(ty::Variance),
  234      TransformTerm(VarianceTermPtr<'a>, VarianceTermPtr<'a>),
  235      InferredTerm(InferredIndex),
  236  }
  237  
  238  impl<'a> fmt::Show for VarianceTerm<'a> {
  239      fn fmt(&self, f: &mut fmt::Formatter) -> fmt::Result {
  240          match *self {
  241              ConstantTerm(c1) => write!(f.buf, "{}", c1),
  242              TransformTerm(v1, v2) => write!(f.buf, "({} \u00D7 {})", v1, v2),
  243              InferredTerm(id) => write!(f.buf, "[{}]", { let InferredIndex(i) = id; i })
  244          }
  245      }
  246  }

  249   * The first pass over the crate simply builds up the set of inferreds.

  251  
  252  struct TermsContext<'a> {
  253      tcx: &'a ty::ctxt,
  254      arena: &'a Arena,
  255  
  256      empty_variances: @ty::ItemVariances,
  257  
  258      // Maps from the node id of a type/generic parameter to the
  259      // corresponding inferred index.
  260      inferred_map: HashMap<ast::NodeId, InferredIndex>,
  261  
  262      // Maps from an InferredIndex to the info for that variable.
  263      inferred_infos: Vec<InferredInfo<'a>> ,
  264  }
  265  
  266  enum ParamKind { TypeParam, RegionParam, SelfParam }
  267  
  268  struct InferredInfo<'a> {
  269      item_id: ast::NodeId,
  270      kind: ParamKind,
  271      index: uint,
  272      param_id: ast::NodeId,
  273      term: VarianceTermPtr<'a>,
  274  }
  275  
  276  fn determine_parameters_to_be_inferred<'a>(tcx: &'a ty::ctxt,
  277                                             arena: &'a mut Arena,
  278                                             krate: &ast::Crate)
  279                                             -> TermsContext<'a> {
  280      let mut terms_cx = TermsContext {
  281          tcx: tcx,
  282          arena: arena,
  283          inferred_map: HashMap::new(),
  284          inferred_infos: Vec::new(),
  285  
  286          // cache and share the variance struct used for items with
  287          // no type/region parameters
  288          empty_variances: @ty::ItemVariances { self_param: None,
  289                                                type_params: OwnedSlice::empty(),
  290                                                region_params: OwnedSlice::empty() }
  291      };
  292  
  293      visit::walk_crate(&mut terms_cx, krate, ());
  294  
  295      terms_cx
  296  }

  298  impl<'a> TermsContext<'a> {
  299      fn add_inferred(&mut self,
  300                      item_id: ast::NodeId,
  301                      kind: ParamKind,
  302                      index: uint,
  303                      param_id: ast::NodeId) {
  304          let inf_index = InferredIndex(self.inferred_infos.len());
  305          let term = self.arena.alloc(|| InferredTerm(inf_index));
  306          self.inferred_infos.push(InferredInfo { item_id: item_id,
  307                                                  kind: kind,
  308                                                  index: index,
  309                                                  param_id: param_id,
  310                                                  term: term });
  311          let newly_added = self.inferred_map.insert(param_id, inf_index);
  312          assert!(newly_added);
  313  
  314          debug!("add_inferred(item_id={}, \
  315                  kind={:?}, \
  316                  index={}, \
  317                  param_id={},
  318                  inf_index={:?})",
  319                  item_id, kind, index, param_id, inf_index);
  320      }
  321  
  322      fn num_inferred(&self) -> uint {
  323          self.inferred_infos.len()
  324      }
  325  }

  327  impl<'a> Visitor<()> for TermsContext<'a> {
  328      fn visit_item(&mut self, item: &ast::Item, _: ()) {
  329          debug!("add_inferreds for item {}", item.repr(self.tcx));
  330  
  331          let inferreds_on_entry = self.num_inferred();
  332  
  333          // NB: In the code below for writing the results back into the
  334          // tcx, we rely on the fact that all inferreds for a particular
  335          // item are assigned continuous indices.
  336          match item.node {
  337              ast::ItemTrait(..) => {
  338                  self.add_inferred(item.id, SelfParam, 0, item.id);
  339              }
  340              _ => { }
  341          }
  342  
  343          match item.node {
  344              ast::ItemEnum(_, ref generics) |
  345              ast::ItemStruct(_, ref generics) |
  346              ast::ItemTrait(ref generics, _, _) => {
  347                  for (i, p) in generics.lifetimes.iter().enumerate() {
  348                      self.add_inferred(item.id, RegionParam, i, p.id);
  349                  }
  350                  for (i, p) in generics.ty_params.iter().enumerate() {
  351                      self.add_inferred(item.id, TypeParam, i, p.id);
  352                  }
  353  
  354                  // If this item has no type or lifetime parameters,
  355                  // then there are no variances to infer, so just
  356                  // insert an empty entry into the variance map.
  357                  // Arguably we could just leave the map empty in this
  358                  // case but it seems cleaner to be able to distinguish
  359                  // "invalid item id" from "item id with no
  360                  // parameters".
  361                  if self.num_inferred() == inferreds_on_entry {
  362                      let newly_added = self.tcx.item_variance_map.borrow_mut().insert(
  363                          ast_util::local_def(item.id),
  364                          self.empty_variances);
  365                      assert!(newly_added);
  366                  }
  367  
  368                  visit::walk_item(self, item, ());
  369              }
  370  
  371              ast::ItemImpl(..) |
  372              ast::ItemStatic(..) |
  373              ast::ItemFn(..) |
  374              ast::ItemMod(..) |
  375              ast::ItemForeignMod(..) |
  376              ast::ItemTy(..) |
  377              ast::ItemMac(..) => {
  378                  visit::walk_item(self, item, ());
  379              }
  380          }
  381      }
  382  }

  384  /**************************************************************************
  385   * Constraint construction and representation
  386   *
  387   * The second pass over the AST determines the set of constraints.
  388   * We walk the set of items and, for each member, generate new constraints.
  389   */
  390  
  391  struct ConstraintContext<'a> {
  392      terms_cx: TermsContext<'a>,
  393  
  394      // These are the def-id of the std::kinds::marker::InvariantType,
  395      // std::kinds::marker::InvariantLifetime, and so on. The arrays
  396      // are indexed by the `ParamKind` (type, lifetime, self). Note
  397      // that there are no marker types for self, so the entries for
  398      // self are always None.
  399      invariant_lang_items: [Option<ast::DefId>, ..3],
  400      covariant_lang_items: [Option<ast::DefId>, ..3],
  401      contravariant_lang_items: [Option<ast::DefId>, ..3],
  402  
  403      // These are pointers to common `ConstantTerm` instances
  404      covariant: VarianceTermPtr<'a>,
  405      contravariant: VarianceTermPtr<'a>,
  406      invariant: VarianceTermPtr<'a>,
  407      bivariant: VarianceTermPtr<'a>,
  408  
  409      constraints: Vec<Constraint<'a>> ,
  410  }
  411  
  412  /// Declares that the variable `decl_id` appears in a location with
  413  /// variance `variance`.
  414  struct Constraint<'a> {
  415      inferred: InferredIndex,
  416      variance: &'a VarianceTerm<'a>,
  417  }
  418  
  419  fn add_constraints_from_crate<'a>(terms_cx: TermsContext<'a>,
  420                                    krate: &ast::Crate)
  421                                    -> ConstraintContext<'a> {
  422      let mut invariant_lang_items = [None, ..3];
  423      let mut covariant_lang_items = [None, ..3];
  424      let mut contravariant_lang_items = [None, ..3];
  425  
  426      covariant_lang_items[TypeParam as uint] =
  427          terms_cx.tcx.lang_items.covariant_type();
  428      covariant_lang_items[RegionParam as uint] =
  429          terms_cx.tcx.lang_items.covariant_lifetime();
  430  
  431      contravariant_lang_items[TypeParam as uint] =
  432          terms_cx.tcx.lang_items.contravariant_type();
  433      contravariant_lang_items[RegionParam as uint] =
  434          terms_cx.tcx.lang_items.contravariant_lifetime();
  435  
  436      invariant_lang_items[TypeParam as uint] =
  437          terms_cx.tcx.lang_items.invariant_type();
  438      invariant_lang_items[RegionParam as uint] =
  439          terms_cx.tcx.lang_items.invariant_lifetime();
  440  
  441      let covariant = terms_cx.arena.alloc(|| ConstantTerm(ty::Covariant));
  442      let contravariant = terms_cx.arena.alloc(|| ConstantTerm(ty::Contravariant));
  443      let invariant = terms_cx.arena.alloc(|| ConstantTerm(ty::Invariant));
  444      let bivariant = terms_cx.arena.alloc(|| ConstantTerm(ty::Bivariant));
  445      let mut constraint_cx = ConstraintContext {
  446          terms_cx: terms_cx,
  447  
  448          invariant_lang_items: invariant_lang_items,
  449          covariant_lang_items: covariant_lang_items,
  450          contravariant_lang_items: contravariant_lang_items,
  451  
  452          covariant: covariant,
  453          contravariant: contravariant,
  454          invariant: invariant,
  455          bivariant: bivariant,
  456          constraints: Vec::new(),
  457      };
  458      visit::walk_crate(&mut constraint_cx, krate, ());
  459      constraint_cx
  460  }
  461  
  462  impl<'a> Visitor<()> for ConstraintContext<'a> {
  463      fn visit_item(&mut self, item: &ast::Item, _: ()) {
  464          let did = ast_util::local_def(item.id);
  465          let tcx = self.terms_cx.tcx;
  466  
  467          match item.node {
  468              ast::ItemEnum(ref enum_definition, _) => {
  469                  // Hack: If we directly call `ty::enum_variants`, it
  470                  // annoyingly takes it upon itself to run off and
  471                  // evaluate the discriminants eagerly (*grumpy* that's
  472                  // not the typical pattern). This results in double
  473                  // error messages because typeck goes off and does
  474                  // this at a later time. All we really care about is
  475                  // the types of the variant arguments, so we just call
  476                  // `ty::VariantInfo::from_ast_variant()` ourselves
  477                  // here, mainly so as to mask the differences between
  478                  // struct-like enums and so forth.
  479                  for &ast_variant in enum_definition.variants.iter() {
  480                      let variant =
  481                          ty::VariantInfo::from_ast_variant(tcx,
  482                                                            ast_variant,
  483                                                            /*discrimant*/ 0);
  484                      for &arg_ty in variant.args.iter() {
  485                          self.add_constraints_from_ty(arg_ty, self.covariant);
  486                      }
  487                  }
  488              }
  489  
  490              ast::ItemStruct(..) => {
  491                  let struct_fields = ty::lookup_struct_fields(tcx, did);
  492                  for field_info in struct_fields.iter() {
  493                      assert_eq!(field_info.id.krate, ast::LOCAL_CRATE);
  494                      let field_ty = ty::node_id_to_type(tcx, field_info.id.node);
  495                      self.add_constraints_from_ty(field_ty, self.covariant);
  496                  }
  497              }
  498  
  499              ast::ItemTrait(..) => {
  500                  let methods = ty::trait_methods(tcx, did);
  501                  for method in methods.iter() {
  502                      self.add_constraints_from_sig(
  503                          &method.fty.sig, self.covariant);
  504                  }
  505              }
  506  
  507              ast::ItemStatic(..) |
  508              ast::ItemFn(..) |
  509              ast::ItemMod(..) |
  510              ast::ItemForeignMod(..) |
  511              ast::ItemTy(..) |
  512              ast::ItemImpl(..) |
  513              ast::ItemMac(..) => {
  514                  visit::walk_item(self, item, ());
  515              }
  516          }
  517      }
  518  }
  519  
  520  impl<'a> ConstraintContext<'a> {
  521      fn tcx(&self) -> &'a ty::ctxt {
  522          self.terms_cx.tcx
  523      }
  524  
  525      fn inferred_index(&self, param_id: ast::NodeId) -> InferredIndex {
  526          match self.terms_cx.inferred_map.find(&param_id) {
  527              Some(&index) => index,
  528              None => {
  529                  self.tcx().sess.bug(format!(
  530                          "No inferred index entry for {}",
  531                          self.tcx().map.node_to_str(param_id)));
  532              }
  533          }
  534      }
  535  
  536      fn declared_variance(&self,
  537                           param_def_id: ast::DefId,
  538                           item_def_id: ast::DefId,
  539                           kind: ParamKind,
  540                           index: uint)
  541                           -> VarianceTermPtr<'a> {
  542          /*!
  543           * Returns a variance term representing the declared variance of
  544           * the type/region parameter with the given id.
  545           */
  546  
  547          assert_eq!(param_def_id.krate, item_def_id.krate);
  548  
  549          if self.invariant_lang_items[kind as uint] == Some(item_def_id) {
  550              self.invariant
  551          } else if self.covariant_lang_items[kind as uint] == Some(item_def_id) {
  552              self.covariant
  553          } else if self.contravariant_lang_items[kind as uint] == Some(item_def_id) {
  554              self.contravariant
  555          } else if param_def_id.krate == ast::LOCAL_CRATE {
  556              // Parameter on an item defined within current crate:
  557              // variance not yet inferred, so return a symbolic
  558              // variance.
  559              let InferredIndex(index) = self.inferred_index(param_def_id.node);
  560              self.terms_cx.inferred_infos.get(index).term
  561          } else {
  562              // Parameter on an item defined within another crate:
  563              // variance already inferred, just look it up.
  564              let variances = ty::item_variances(self.tcx(), item_def_id);
  565              let variance = match kind {
  566                  SelfParam => variances.self_param.unwrap(),
  567                  TypeParam => *variances.type_params.get(index),
  568                  RegionParam => *variances.region_params.get(index),
  569              };
  570              self.constant_term(variance)
  571          }
  572      }
  573  
  574      fn add_constraint(&mut self,
  575                        InferredIndex(index): InferredIndex,
  576                        variance: VarianceTermPtr<'a>) {
  577          debug!("add_constraint(index={}, variance={})",
  578                  index, variance.to_str());
  579          self.constraints.push(Constraint { inferred: InferredIndex(index),
  580                                             variance: variance });
  581      }
  582  
  583      fn contravariant(&mut self,
  584                       variance: VarianceTermPtr<'a>)
  585                       -> VarianceTermPtr<'a> {
  586          self.xform(variance, self.contravariant)
  587      }
  588  
  589      fn invariant(&mut self,
  590                   variance: VarianceTermPtr<'a>)
  591                   -> VarianceTermPtr<'a> {
  592          self.xform(variance, self.invariant)
  593      }
  594  
  595      fn constant_term(&self, v: ty::Variance) -> VarianceTermPtr<'a> {
  596          match v {
  597              ty::Covariant => self.covariant,
  598              ty::Invariant => self.invariant,
  599              ty::Contravariant => self.contravariant,
  600              ty::Bivariant => self.bivariant,
  601          }
  602      }
  603  
  604      fn xform(&mut self,
  605               v1: VarianceTermPtr<'a>,
  606               v2: VarianceTermPtr<'a>)
  607               -> VarianceTermPtr<'a> {
  608          match (*v1, *v2) {
  609              (_, ConstantTerm(ty::Covariant)) => {
  610                  // Applying a "covariant" transform is always a no-op
  611                  v1
  612              }
  613  
  614              (ConstantTerm(c1), ConstantTerm(c2)) => {
  615                  self.constant_term(c1.xform(c2))
  616              }
  617  
  618              _ => {
  619                  self.terms_cx.arena.alloc(|| TransformTerm(v1, v2))
  620              }
  621          }
  622      }
  623  
  624      /// Adds constraints appropriate for an instance of `ty` appearing
  625      /// in a context with ambient variance `variance`
  626      fn add_constraints_from_ty(&mut self,
  627                                 ty: ty::t,
  628                                 variance: VarianceTermPtr<'a>) {
  629          debug!("add_constraints_from_ty(ty={})", ty.repr(self.tcx()));
  630  
  631          match ty::get(ty).sty {
  632              ty::ty_nil | ty::ty_bot | ty::ty_bool |
  633              ty::ty_char | ty::ty_int(_) | ty::ty_uint(_) |
  634              ty::ty_float(_) => {
  635                  /* leaf type -- noop */
  636              }
  637  
  638              ty::ty_rptr(region, ref mt) => {
  639                  let contra = self.contravariant(variance);
  640                  self.add_constraints_from_region(region, contra);
  641                  self.add_constraints_from_mt(mt, variance);
  642              }
  643  
  644              ty::ty_str(vstore) => {
  645                  self.add_constraints_from_vstore(vstore, variance);
  646              }
  647  
  648              ty::ty_vec(ref mt, vstore) => {
  649                  self.add_constraints_from_vstore(vstore, variance);
  650                  self.add_constraints_from_mt(mt, variance);
  651              }
  652  
  653              ty::ty_uniq(typ) | ty::ty_box(typ) => {
  654                  self.add_constraints_from_ty(typ, variance);
  655              }
  656  
  657              ty::ty_ptr(ref mt) => {
  658                  self.add_constraints_from_mt(mt, variance);
  659              }
  660  
  661              ty::ty_tup(ref subtys) => {
  662                  for &subty in subtys.iter() {
  663                      self.add_constraints_from_ty(subty, variance);
  664                  }
  665              }
  666  
  667              ty::ty_enum(def_id, ref substs) |
  668              ty::ty_struct(def_id, ref substs) => {
  669                  let item_type = ty::lookup_item_type(self.tcx(), def_id);
  670                  self.add_constraints_from_substs(def_id, &item_type.generics,
  671                                                   substs, variance);
  672              }
  673  
  674              ty::ty_trait(~ty::TyTrait { def_id, ref substs, .. }) => {
  675                  let trait_def = ty::lookup_trait_def(self.tcx(), def_id);
  676                  self.add_constraints_from_substs(def_id, &trait_def.generics,
  677                                                   substs, variance);
  678              }
  679  
  680              ty::ty_param(ty::param_ty { def_id: ref def_id, .. }) => {
  681                  assert_eq!(def_id.krate, ast::LOCAL_CRATE);
  682                  match self.terms_cx.inferred_map.find(&def_id.node) {
  683                      Some(&index) => {
  684                          self.add_constraint(index, variance);
  685                      }
  686                      None => {
  687                          // We do not infer variance for type parameters
  688                          // declared on methods. They will not be present
  689                          // in the inferred_map.
  690                      }
  691                  }
  692              }
  693  
  694              ty::ty_self(ref def_id) => {
  695                  assert_eq!(def_id.krate, ast::LOCAL_CRATE);
  696                  let index = self.inferred_index(def_id.node);
  697                  self.add_constraint(index, variance);
  698              }
  699  
  700              ty::ty_bare_fn(ty::BareFnTy { sig: ref sig, .. }) => {
  701                  self.add_constraints_from_sig(sig, variance);
  702              }
  703  
  704              ty::ty_closure(~ty::ClosureTy { sig: ref sig, region, .. }) => {
  705                  let contra = self.contravariant(variance);
  706                  self.add_constraints_from_region(region, contra);
  707                  self.add_constraints_from_sig(sig, variance);
  708              }
  709  
  710              ty::ty_infer(..) | ty::ty_err | ty::ty_unboxed_vec(..) => {
  711                  self.tcx().sess.bug(
  712                      format!("unexpected type encountered in \
  713                              variance inference: {}",
  714                              ty.repr(self.tcx())));
  715              }
  716          }
  717      }
  718  
  719      /// Adds constraints appropriate for a vector with vstore `vstore`
  720      /// appearing in a context with ambient variance `variance`
  721      fn add_constraints_from_vstore(&mut self,
  722                                     vstore: ty::vstore,
  723                                     variance: VarianceTermPtr<'a>) {
  724          match vstore {
  725              ty::vstore_slice(r) => {
  726                  let contra = self.contravariant(variance);
  727                  self.add_constraints_from_region(r, contra);
  728              }
  729  
  730              ty::vstore_fixed(_) | ty::vstore_uniq => {
  731              }
  732          }
  733      }
  734  
  735      /// Adds constraints appropriate for a nominal type (enum, struct,
  736      /// object, etc) appearing in a context with ambient variance `variance`
  737      fn add_constraints_from_substs(&mut self,
  738                                     def_id: ast::DefId,
  739                                     generics: &ty::Generics,
  740                                     substs: &ty::substs,
  741                                     variance: VarianceTermPtr<'a>) {
  742          debug!("add_constraints_from_substs(def_id={:?})", def_id);
  743  
  744          for (i, p) in generics.type_param_defs().iter().enumerate() {
  745              let variance_decl =
  746                  self.declared_variance(p.def_id, def_id, TypeParam, i);
  747              let variance_i = self.xform(variance, variance_decl);
  748              self.add_constraints_from_ty(*substs.tps.get(i), variance_i);
  749          }
  750  
  751          match substs.regions {
  752              ty::ErasedRegions => {}
  753              ty::NonerasedRegions(ref rps) => {
  754                  for (i, p) in generics.region_param_defs().iter().enumerate() {
  755                      let variance_decl =
  756                          self.declared_variance(p.def_id, def_id, RegionParam, i);
  757                      let variance_i = self.xform(variance, variance_decl);
  758                      self.add_constraints_from_region(*rps.get(i), variance_i);
  759                  }
  760              }
  761          }
  762      }
  763  
  764      /// Adds constraints appropriate for a function with signature
  765      /// `sig` appearing in a context with ambient variance `variance`
  766      fn add_constraints_from_sig(&mut self,
  767                                  sig: &ty::FnSig,
  768                                  variance: VarianceTermPtr<'a>) {
  769          let contra = self.contravariant(variance);
  770          for &input in sig.inputs.iter() {
  771              self.add_constraints_from_ty(input, contra);
  772          }
  773          self.add_constraints_from_ty(sig.output, variance);
  774      }
  775  
  776      /// Adds constraints appropriate for a region appearing in a
  777      /// context with ambient variance `variance`
  778      fn add_constraints_from_region(&mut self,
  779                                     region: ty::Region,
  780                                     variance: VarianceTermPtr<'a>) {
  781          match region {
  782              ty::ReEarlyBound(param_id, _, _) => {
  783                  let index = self.inferred_index(param_id);
  784                  self.add_constraint(index, variance);
  785              }
  786  
  787              ty::ReStatic => { }
  788  
  789              ty::ReLateBound(..) => {
  790                  // We do not infer variance for region parameters on
  791                  // methods or in fn types.
  792              }
  793  
  794              ty::ReFree(..) | ty::ReScope(..) | ty::ReInfer(..) |
  795              ty::ReEmpty => {
  796                  // We don't expect to see anything but 'static or bound
  797                  // regions when visiting member types or method types.
  798                  self.tcx().sess.bug(format!("unexpected region encountered in \

  855          // number of constraints (each variable can change values at most
