   11  /*!
   12  
   13  # Method lookup
   14  
   15  Method lookup can be rather complex due to the interaction of a number
   16  of factors, such as self types, autoderef, trait lookup, etc.  The
   17  algorithm is divided into two parts: candidate collection and
   18  candidate selection.
   19  
   20  ## Candidate collection
   21  
   22  A `Candidate` is a method item that might plausibly be the method
   23  being invoked.  Candidates are grouped into two kinds, inherent and
   24  extension.  Inherent candidates are those that are derived from the
   25  type of the receiver itself.  So, if you have a receiver of some
   26  nominal type `Foo` (e.g., a struct), any methods defined within an
   27  impl like `impl Foo` are inherent methods.  Nothing needs to be
   28  imported to use an inherent method, they are associated with the type
   29  itself (note that inherent impls can only be defined in the same
   30  module as the type itself).
   31  
   32  Inherent candidates are not always derived from impls.  If you have a
   33  trait instance, such as a value of type `@ToStr`, then the trait
   34  methods (`to_str()`, in this case) are inherently associated with it.
   35  Another case is type parameters, in which case the methods of their
   36  bounds are inherent.
   37  
   38  Extension candidates are derived from imported traits.  If I have the
   39  trait `ToStr` imported, and I call `to_str()` on a value of type `T`,
   40  then we will go off to find out whether there is an impl of `ToStr`
   41  for `T`.  These kinds of method calls are called "extension methods".
   42  They can be defined in any module, not only the one that defined `T`.
   43  Furthermore, you must import the trait to call such a method.
   44  
   45  For better or worse, we currently give weight to inherent methods over
   46  extension methods during candidate selection (below).
   47  
   48  ## Candidate selection
   49  
   50  Once we know the set of candidates, we can go off and try to select
   51  which one is actually being called.  We do this by taking the type of
   52  the receiver, let's call it R, and checking whether it matches against
   53  the expected receiver type for each of the collected candidates.  We
   54  first check for inherent candidates and see whether we get exactly one
   55  match (zero means keep searching, more than one is an error).  If so,
   56  we return that as the candidate.  Otherwise we search the extension
   57  candidates in the same way.
   58  
   59  If find no matching candidate at all, we proceed to auto-deref the
   60  receiver type and search again.  We keep doing that until we cannot
   61  auto-deref any longer.  At each step, we also check for candidates
   62  based on "autoptr", which if the current type is `T`, checks for `&mut
   63  T`, `&const T`, and `&T` receivers.  Finally, at the very end, we will
   64  also try autoslice, which converts `~[]` to `&[]` (there is no point
   65  at trying autoslice earlier, because no autoderefable type is also
   66  sliceable).
   67  
   68  ## Why two phases?
   69  
   70  You might wonder why we first collect the candidates and then select.
   71  Both the inherent candidate collection and the candidate selection
   72  proceed by progressively deref'ing the receiver type, after all.  The
   73  answer is that two phases are needed to elegantly deal with explicit
   74  self.  After all, if there is an impl for the type `Foo`, it can
   75  define a method with the type `~self`, which means that it expects a
   76  receiver of type `~Foo`.  If we have a receiver of type `~Foo`, but we
   77  waited to search for that impl until we have deref'd the `~` away and
   78  obtained the type `Foo`, we would never match this method.
   79  
   80  */

  120  pub fn lookup<'a>(
  121          fcx: &'a FnCtxt<'a>,
  122  
  123          // In a call `a.b::<X, Y, ...>(...)`:
  124          expr: &ast::Expr,                   // The expression `a.b(...)`.
  125          self_expr: &'a ast::Expr,           // The expression `a`.
  126          m_name: ast::Name,                  // The name `b`.
  127          self_ty: ty::t,                     // The type of `a`.
  128          supplied_tps: &'a [ty::t],          // The list of types X, Y, ... .
  129          deref_args: check::DerefArgs,       // Whether we autopointer first.
  130          check_traits: CheckTraitsFlag,      // Whether we check traits only.
  131          autoderef_receiver: AutoderefReceiverFlag)
  132       -> Option<MethodCallee> {
  133      let mut lcx = LookupContext {
  134          fcx: fcx,
  135          span: expr.span,
  136          self_expr: Some(self_expr),
  137          m_name: m_name,
  138          supplied_tps: supplied_tps,
  139          impl_dups: HashSet::new(),
  140          inherent_candidates: Vec::new(),
  141          extension_candidates: Vec::new(),
  142          deref_args: deref_args,
  143          check_traits: check_traits,
  144          autoderef_receiver: autoderef_receiver,
  145      };
  146  
  147      debug!("method lookup(self_ty={}, expr={}, self_expr={})",
  148             self_ty.repr(fcx.tcx()), expr.repr(fcx.tcx()),
  149             self_expr.repr(fcx.tcx()));
  150  
  151      debug!("searching inherent candidates");
  152      lcx.push_inherent_candidates(self_ty);
  153      let mme = lcx.search(self_ty);
  154      if mme.is_some() {
  155          return mme;
  156      }
  157  
  158      debug!("searching extension candidates");
  159      lcx.reset_candidates();
  160      lcx.push_bound_candidates(self_ty, None);
  161      lcx.push_extension_candidates(expr.id);
  162      lcx.search(self_ty)
  163  }

  201  // Determine the index of a method in the list of all methods belonging
  202  // to a trait and its supertraits.
  203  fn get_method_index(tcx: &ty::ctxt,
  204                      trait_ref: &TraitRef,
  205                      subtrait: @TraitRef,
  206                      n_method: uint) -> uint {
  207      // We need to figure the "real index" of the method in a
  208      // listing of all the methods of an object. We do this by
  209      // iterating down the supertraits of the object's trait until
  210      // we find the trait the method came from, counting up the
  211      // methods from them.
  212      let mut method_count = 0;
  213      ty::each_bound_trait_and_supertraits(tcx, &[subtrait], |bound_ref| {
  214          if bound_ref.def_id == trait_ref.def_id { false }
  215              else {
  216              method_count += ty::trait_methods(tcx, bound_ref.def_id).len();
  217              true
  218          }
  219      });
  220      method_count + n_method
  221  }

  223  fn construct_transformed_self_ty_for_object(
  224      tcx: &ty::ctxt,
  225      span: Span,
  226      trait_def_id: ast::DefId,
  227      rcvr_substs: &ty::substs,
  228      method_ty: &ty::Method)
  229      -> ty::t {
  230      /*!
  231          * This is a bit tricky. We have a match against a trait method
  232          * being invoked on an object, and we want to generate the
  233          * self-type. As an example, consider a trait
  234          *
  235          *     trait Foo {
  236          *         fn r_method<'a>(&'a self);
  237          *         fn u_method(~self);
  238          *     }
  239          *
  240          * Now, assuming that `r_method` is being called, we want the
  241          * result to be `&'a Foo`. Assuming that `u_method` is being
  242          * called, we want the result to be `~Foo`. Of course,
  243          * this transformation has already been done as part of
  244          * `method_ty.fty.sig.inputs[0]`, but there the type
  245          * is expressed in terms of `Self` (i.e., `&'a Self`, `~Self`).
  246          * Because objects are not standalone types, we can't just substitute
  247          * `s/Self/Foo/`, so we must instead perform this kind of hokey
  248          * match below.
  249          */
  250  
  251      let substs = ty::substs {regions: rcvr_substs.regions.clone(),
  252                                  self_ty: None,
  253                                  tps: rcvr_substs.tps.clone()};
  254      match method_ty.explicit_self {
  255          ast::SelfStatic => {
  256              tcx.sess.span_bug(span, "static method for object type receiver");
  257          }
  258          ast::SelfValue => {
  259              ty::mk_err() // error reported in `enforce_object_limitations()`
  260          }
  261          ast::SelfRegion(..) | ast::SelfUniq => {
  262              let transformed_self_ty = *method_ty.fty.sig.inputs.get(0);
  263              match ty::get(transformed_self_ty).sty {
  264                  ty::ty_rptr(r, mt) => { // must be SelfRegion
  265                      let r = r.subst(tcx, &substs); // handle Early-Bound lifetime
  266                      ty::mk_trait(tcx, trait_def_id, substs,
  267                                   RegionTraitStore(r), mt.mutbl,
  268                                   ty::EmptyBuiltinBounds())
  269                  }
  270                  ty::ty_uniq(_) => { // must be SelfUniq
  271                      ty::mk_trait(tcx, trait_def_id, substs,
  272                                   UniqTraitStore, ast::MutImmutable,
  273                                   ty::EmptyBuiltinBounds())
  274                  }
  275                  _ => {
  276                      tcx.sess.span_bug(span,
  277                          format!("'impossible' transformed_self_ty: {}",
  278                                  transformed_self_ty.repr(tcx)));
  279                  }
  280              }
  281          }
  282      }
  283  }

  317  /// This type represents the conditions under which the receiver is
  318  /// considered to "match" a given method candidate. Typically the test
  319  /// is whether the receiver is of a particular type. However, this
  320  /// type is the type of the receiver *after accounting for the
  321  /// method's self type* (e.g., if the method is an `~self` method, we
  322  /// have *already verified* that the receiver is of some type `~T` and
  323  /// now we must check that the type `T` is correct).  Unfortunately,
  324  /// because traits are not types, this is a pain to do.
  325  #[deriving(Clone)]
  326  pub enum RcvrMatchCondition {
  327      RcvrMatchesIfObject(ast::DefId),
  328      RcvrMatchesIfSubtype(ty::t)
  329  }

  331  impl<'a> LookupContext<'a> {
  332      fn search(&self, self_ty: ty::t) -> Option<MethodCallee> {
  333          let span = self.self_expr.map_or(self.span, |e| e.span);
  334          let self_expr_id = self.self_expr.map(|e| e.id);
  335          let (self_ty, autoderefs, result) =
  336              check::autoderef(
  337                  self.fcx, span, self_ty, self_expr_id, PreferMutLvalue,
  338                  |self_ty, autoderefs| self.search_step(self_ty, autoderefs));
  339  
  340          match result {
  341              Some(Some(result)) => Some(result),
  342              _ => {
  343                  if self.is_overloaded_deref() {
  344                      // If we are searching for an overloaded deref, no
  345                      // need to try coercing a `~[T]` to an `&[T]` and
  346                      // searching for an overloaded deref on *that*.
  347                      None
  348                  } else {
  349                      self.search_for_autosliced_method(self_ty, autoderefs)
  350                  }
  351              }
  352          }
  353      }

  355      fn search_step(&self,
  356                     self_ty: ty::t,
  357                     autoderefs: uint)
  358                     -> Option<Option<MethodCallee>> {
  359          debug!("search_step: self_ty={} autoderefs={}",
  360                 self.ty_to_str(self_ty), autoderefs);
  361  
  362          match self.deref_args {
  363              check::DontDerefArgs => {
  364                  match self.search_for_autoderefd_method(self_ty, autoderefs) {
  365                      Some(result) => return Some(Some(result)),
  366                      None => {}
  367                  }
  368  
  369                  match self.search_for_autoptrd_method(self_ty, autoderefs) {
  370                      Some(result) => return Some(Some(result)),
  371                      None => {}
  372                  }
  373              }
  374              check::DoDerefArgs => {
  375                  match self.search_for_autoptrd_method(self_ty, autoderefs) {
  376                      Some(result) => return Some(Some(result)),
  377                      None => {}
  378                  }
  379  
  380                  match self.search_for_autoderefd_method(self_ty, autoderefs) {
  381                      Some(result) => return Some(Some(result)),
  382                      None => {}
  383                  }

  407      fn push_inherent_candidates(&mut self, self_ty: ty::t) {

  410           * `self.inherent_candidates`.  See comment at the start of
  411           * the file.  To find the inherent candidates, we repeatedly
  412           * deref the self-ty to find the "base-type".  So, for
  413           * example, if the receiver is @@C where `C` is a struct type,
  414           * we'll want to find the inherent impls for `C`.
  415           */

  420                  ty_trait(~TyTrait { def_id, ref substs, .. }) => {

  427                      }
  428                  }

  430              }

  432              // Don't autoderef if we aren't supposed to.
  433              if self.autoderef_receiver == DontAutoderefReceiver {

  435              } else {

  488      fn push_inherent_candidates_from_object(&mut self,

  501          // to the `Self` type. Substituting ty_err here allows
  502          // compiler to soldier on.

  505          // for Self. (fix)

  651      fn push_candidates_from_impl(&mut self,
  652                                   impl_info: &ty::Impl,
  653                                   is_extension: bool) {

  655              return; // already visited
  656          }

  659                 token::get_name(self.m_name),

  662                                  .map(|m| m.ident)

  664                                  .repr(self.tcx()));

  668                             .iter()
  669                             .position(|m| m.ident.name == self.m_name) {

  671                  None => { return; } // No method with the right name.

  681          let vcx = self.fcx.vtable_context();

  683              substs: impl_substs,

  691          };

  694              rcvr_match_condition: RcvrMatchesIfSubtype(impl_ty),

  704      fn search_for_autoderefd_method(&self,
  705                                      self_ty: ty::t,
  706                                      autoderefs: uint)
  707                                      -> Option<MethodCallee> {

  709              self.consider_reborrow(self_ty, autoderefs);

  712          // added to the expression from the outside context, so we do not store
  713          // an explicit adjustment, but rather we hardwire the single deref

  721              None => None,

  724                         adjustment {:?}", adjustment);

  726                      Some((self_expr_id, adj)) => {
  727                          self.fcx.write_adjustment(self_expr_id, adj);
  728                      }

  730                  }

  732              }
  733          }
  734      }

  736      fn consider_reborrow(&self,
  737                           self_ty: ty::t,
  738                           autoderefs: uint)
  739                           -> (ty::t, ty::AutoDerefRef) {
  740          /*!
  741           * In the event that we are invoking a method with a receiver
  742           * of a borrowed type like `&T`, `&mut T`, or `&mut [T]`,
  743           * we will "reborrow" the receiver implicitly.  For example, if
  744           * you have a call `r.inc()` and where `r` has type `&mut T`,
  745           * then we treat that like `(&mut *r).inc()`.  This avoids
  746           * consuming the original pointer.
  747           *
  748           * You might think that this would be a natural byproduct of
  749           * the auto-deref/auto-ref process.  This is true for `~T`
  750           * but not for an `&mut T` receiver.  With `~T`, we would
  751           * begin by testing for methods with a self type `~T`,
  752           * then autoderef to `T`, then autoref to `&mut T`.  But with
  753           * an `&mut T` receiver the process begins with `&mut T`, only
  754           * without any autoadjustments.
  755           */
  756  
  757          let tcx = self.tcx();
  758          return match ty::get(self_ty).sty {
  759              ty::ty_rptr(_, self_mt) if default_method_hack(self_mt) => {
  760                  (self_ty,
  761                   ty::AutoDerefRef {
  762                       autoderefs: autoderefs,
  763                       autoref: None})
  764              }
  765              ty::ty_rptr(_, self_mt) => {
  766                  let region =
  767                      self.infcx().next_region_var(infer::Autoref(self.span));
  768                  (ty::mk_rptr(tcx, region, self_mt),
  769                   ty::AutoDerefRef {
  770                       autoderefs: autoderefs+1,
  771                       autoref: Some(ty::AutoPtr(region, self_mt.mutbl))})
  772              }
  773              ty::ty_vec(self_mt, vstore_slice(_)) => {
  774                  let region =
  775                      self.infcx().next_region_var(infer::Autoref(self.span));
  776                  (ty::mk_vec(tcx, self_mt, vstore_slice(region)),
  777                   ty::AutoDerefRef {
  778                       autoderefs: autoderefs,
  779                       autoref: Some(ty::AutoBorrowVec(region, self_mt.mutbl))})
  780              }
  781              ty::ty_trait(~ty::TyTrait {
  782                  def_id, ref substs, store: ty::RegionTraitStore(_), mutability: mutbl, bounds
  783              }) => {
  784                  let region =
  785                      self.infcx().next_region_var(infer::Autoref(self.span));
  786                  (ty::mk_trait(tcx, def_id, substs.clone(),
  787                                ty::RegionTraitStore(region),
  788                                mutbl, bounds),
  789                   ty::AutoDerefRef {
  790                       autoderefs: autoderefs,
  791                       autoref: Some(ty::AutoBorrowObj(region, mutbl))})
  792              }
  793              _ => {
  794                  (self_ty,
  795                   ty::AutoDerefRef {
  796                       autoderefs: autoderefs,
  797                       autoref: None})
  798              }

 1065          // 1. Substitute values for any type/lifetime parameters from the impl and
 1066          //    method declaration into the method type. This is the function type
 1067          //    before it is called; it may still include late bound region variables.
 1068          // 2. Instantiate any late bound lifetime parameters in the method itself
 1069          //    with fresh region variables.
 1070  
 1071          let tcx = self.tcx();
 1072  
 1073          debug!("confirm_candidate(rcvr_ty={}, candidate={})",
 1074                 self.ty_to_str(rcvr_ty),
 1075                 candidate.repr(self.tcx()));

 1077          self.enforce_object_limitations(candidate);
 1078          self.enforce_drop_trait_limitations(candidate);
 1079  
 1080          // static methods should never have gotten this far:
 1081          assert!(candidate.method_ty.explicit_self != SelfStatic);
 1082  
 1083          // Determine the values for the generic parameters of the method.
 1084          // If they were not explicitly supplied, just construct fresh
 1085          // variables.

 1093                      self.span,
