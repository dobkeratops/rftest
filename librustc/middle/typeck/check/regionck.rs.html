   11  /*!
   12  
   13  The region check is a final pass that runs over the AST after we have
   14  inferred the type constraints but before we have actually finalized
   15  the types.  Its purpose is to embed a variety of region constraints.
   16  Inserting these constraints as a separate pass is good because (1) it
   17  localizes the code that has to do with region inference and (2) often
   18  we cannot know what constraints are needed until the basic types have
   19  been inferred.
   20  
   21  ### Interaction with the borrow checker
   22  
   23  In general, the job of the borrowck module (which runs later) is to
   24  check that all soundness criteria are met, given a particular set of
   25  regions. The job of *this* module is to anticipate the needs of the
   26  borrow checker and infer regions that will satisfy its requirements.
   27  It is generally true that the inference doesn't need to be sound,
   28  meaning that if there is a bug and we inferred bad regions, the borrow
   29  checker should catch it. This is not entirely true though; for
   30  example, the borrow checker doesn't check subtyping, and it doesn't
   31  check that region pointers are always live when they are used. It
   32  might be worthwhile to fix this so that borrowck serves as a kind of
   33  verification step -- that would add confidence in the overall
   34  correctness of the compiler, at the cost of duplicating some type
   35  checks and effort.
   36  
   37  ### Inferring the duration of borrows, automatic and otherwise
   38  
   39  Whenever we introduce a borrowed pointer, for example as the result of
   40  a borrow expression `let x = &data`, the lifetime of the pointer `x`
   41  is always specified as a region inference variable. `regionck` has the
   42  job of adding constraints such that this inference variable is as
   43  narrow as possible while still accommodating all uses (that is, every
   44  dereference of the resulting pointer must be within the lifetime).
   45  
   46  #### Reborrows
   47  
   48  Generally speaking, `regionck` does NOT try to ensure that the data
   49  `data` will outlive the pointer `x`. That is the job of borrowck.  The
   50  one exception is when "re-borrowing" the contents of another borrowed
   51  pointer. For example, imagine you have a borrowed pointer `b` with
   52  lifetime L1 and you have an expression `&*b`. The result of this
   53  expression will be another borrowed pointer with lifetime L2 (which is
   54  an inference variable). The borrow checker is going to enforce the
   55  constraint that L2 < L1, because otherwise you are re-borrowing data
   56  for a lifetime larger than the original loan.  However, without the
   57  routines in this module, the region inferencer would not know of this
   58  dependency and thus it might infer the lifetime of L2 to be greater
   59  than L1 (issue #3148).
   60  
   61  There are a number of troublesome scenarios in the tests
   62  `region-dependent-*.rs`, but here is one example:
   63  
   64      struct Foo { i: int }
   65      struct Bar { foo: Foo  }
   66      fn get_i(x: &'a Bar) -> &'a int {
   67         let foo = &x.foo; // Lifetime L1
   68         &foo.i            // Lifetime L2
   69      }
   70  
   71  Note that this comes up either with `&` expressions, `ref`
   72  bindings, and `autorefs`, which are the three ways to introduce
   73  a borrow.
   74  
   75  The key point here is that when you are borrowing a value that
   76  is "guaranteed" by a borrowed pointer, you must link the
   77  lifetime of that borrowed pointer (L1, here) to the lifetime of
   78  the borrow itself (L2).  What do I mean by "guaranteed" by a
   79  borrowed pointer? I mean any data that is reached by first
   80  dereferencing a borrowed pointer and then either traversing
   81  interior offsets or owned pointers.  We say that the guarantor
   82  of such data it the region of the borrowed pointer that was
   83  traversed.  This is essentially the same as the ownership
   84  relation, except that a borrowed pointer never owns its
   85  contents.
   86  
   87  ### Inferring borrow kinds for upvars
   88  
   89  Whenever there is a closure expression, we need to determine how each
   90  upvar is used. We do this by initially assigning each upvar an
   91  immutable "borrow kind" (see `ty::BorrowKind` for details) and then
   92  "escalating" the kind as needed. The borrow kind proceeds according to
   93  the following lattice:
   94  
   95      ty::ImmBorrow -> ty::UniqueImmBorrow -> ty::MutBorrow
   96  
   97  So, for example, if we see an assignment `x = 5` to an upvar `x`, we
   98  will promote its borrow kind to mutable borrow. If we see an `&mut x`
   99  we'll do the same. Naturally, this applies not just to the upvar, but
  100  to everything owned by `x`, so the result is the same for something
  101  like `x.f = 5` and so on (presuming `x` is not a borrowed pointer to a
  102  struct). These adjustments are performed in
  103  `adjust_upvar_borrow_kind()` (you can trace backwards through the code
  104  from there).
  105  
  106  The fact that we are inferring borrow kinds as we go results in a
  107  semi-hacky interaction with mem-categorization. In particular,
  108  mem-categorization will query the current borrow kind as it
  109  categorizes, and we'll return the *current* value, but this may get
  110  adjusted later. Therefore, in this module, we generally ignore the
  111  borrow kind (and derived mutabilities) that are returned from
  112  mem-categorization, since they may be inaccurate. (Another option
  113  would be to use a unification scheme, where instead of returning a
  114  concrete borrow kind like `ty::ImmBorrow`, we return a
  115  `ty::InferBorrow(upvar_id)` or something like that, but this would
  116  then mean that all later passes would have to check for these figments
  117  and report an error, and it just seems like more mess in the end.)
  118  
  119  */

  165  fn region_of_def(fcx: &FnCtxt, def: ast::Def) -> ty::Region {
  166      /*!
  167       * Returns the validity region of `def` -- that is, how long
  168       * is `def` valid?
  169       */
  170  
  171      let tcx = fcx.tcx();
  172      match def {
  173          DefLocal(node_id, _) | DefArg(node_id, _) |
  174          DefBinding(node_id, _) => {
  175              tcx.region_maps.var_region(node_id)
  176          }
  177          DefUpvar(_, subdef, closure_id, body_id) => {
  178              match ty::ty_closure_sigil(fcx.node_ty(closure_id)) {
  179                  BorrowedSigil => region_of_def(fcx, *subdef),
  180                  ManagedSigil | OwnedSigil => ReScope(body_id)
  181              }
  182          }
  183          _ => {
  184              tcx.sess.bug(format!("unexpected def in region_of_def: {:?}",
  185                                def))
  186          }
  187      }
  188  }

  201      pub fn resolve_type(&self, unresolved_ty: ty::t) -> ty::t {
  202          /*!
  203           * Try to resolve the type for the given node, returning
  204           * t_err if an error results.  Note that we never care
  205           * about the details of the error, the same error will be
  206           * detected and reported in the writeback phase.
  207           *
  208           * Note one important point: we do not attempt to resolve
  209           * *region variables* here.  This is because regionck is
  210           * essentially adding constraints to those region variables
  211           * and so may yet influence how they are resolved.
  212           *
  213           * Consider this silly example:
  214           *
  215           *     fn borrow(x: &int) -> &int {x}
  216           *     fn foo(x: @int) -> int {  // block: B
  217           *         let b = borrow(x);    // region: <R0>
  218           *         *b
  219           *     }
  220           *
  221           * Here, the region of `b` will be `<R0>`.  `<R0>` is
  222           * constrainted to be some subregion of the block B and some
  223           * superregion of the call.  If we forced it now, we'd choose
  224           * the smaller region (the call).  But that would make the *b
  225           * illegal.  Since we don't resolve, the type of b will be
  226           * `&<R0>.int` and then `*b` will require that `<R0>` be
  227           * bigger than the let and the `*b` expression, so we will
  228           * effectively resolve `<R0>` to be the block B.
  229           */
  230          match resolve_type(self.fcx.infcx(), unresolved_ty,
  231                             resolve_and_force_all_but_regions) {
  232              Ok(t) => t,
  233              Err(_) => ty::mk_err()
  234          }
  235      }

  239          let t = self.fcx.node_ty(id);

  243      fn resolve_method_type(&self, method_call: MethodCall) -> Option<ty::t> {
  244          let method_ty = self.fcx.inh.method_map.borrow()
  245                              .find(&method_call).map(|method| method.ty);
  246          method_ty.map(|method_ty| self.resolve_type(method_ty))

  249      /// Try to resolve the type for the given node.
  250      pub fn resolve_expr_type_adjusted(&mut self, expr: &ast::Expr) -> ty::t {
  251          let ty_unadjusted = self.resolve_node_type(expr.id);
  252          if ty::type_is_error(ty_unadjusted) || ty::type_is_bot(ty_unadjusted) {
  253              ty_unadjusted
  254          } else {
  255              let tcx = self.fcx.tcx();
  256              let adjustment = self.fcx.inh.adjustments.borrow().find_copy(&expr.id);
  257              ty::adjust_ty(tcx, expr.span, expr.id, ty_unadjusted, adjustment,
  258                            |method_call| self.resolve_method_type(method_call))
  259          }
  260      }
  261  }

  362  fn constrain_bindings_in_pat(pat: &ast::Pat, rcx: &mut Rcx) {
  363      let tcx = rcx.fcx.tcx();
  364      debug!("regionck::visit_pat(pat={})", pat.repr(tcx));
  365      pat_util::pat_bindings(tcx.def_map, pat, |_, id, span, _| {
  366          // If we have a variable that contains region'd data, that
  367          // data will be accessible from anywhere that the variable is
  368          // accessed. We must be wary of loops like this:
  369          //
  370          //     // from src/test/compile-fail/borrowck-lend-flow.rs
  371          //     let mut v = ~3, w = ~4;
  372          //     let mut x = &mut w;
  373          //     loop {
  374          //         **x += 1;   // (2)
  375          //         borrow(v);  //~ ERROR cannot borrow
  376          //         x = &mut v; // (1)
  377          //     }
  378          //
  379          // Typically, we try to determine the region of a borrow from
  380          // those points where it is dereferenced. In this case, one
  381          // might imagine that the lifetime of `x` need only be the
  382          // body of the loop. But of course this is incorrect because
  383          // the pointer that is created at point (1) is consumed at
  384          // point (2), meaning that it must be live across the loop
  385          // iteration. The easiest way to guarantee this is to require
  386          // that the lifetime of any regions that appear in a
  387          // variable's type enclose at least the variable's scope.
  388  
  389          let var_region = tcx.region_maps.var_region(id);
  390          constrain_regions_in_type_of_node(
  391              rcx, id, var_region,
  392              infer::BindingTypeIsNotValidAtDecl(span));
  393      })
  394  }

  396  fn visit_expr(rcx: &mut Rcx, expr: &ast::Expr) {
  397      debug!("regionck::visit_expr(e={}, repeating_scope={:?})",
  398             expr.repr(rcx.fcx.tcx()), rcx.repeating_scope);
  399  
  400      let method_call = MethodCall::expr(expr.id);
  401      let has_method_map = rcx.fcx.inh.method_map.borrow().contains_key(&method_call);
  402  
  403      // Check any autoderefs or autorefs that appear.
  404      for &adjustment in rcx.fcx.inh.adjustments.borrow().find(&expr.id).iter() {
  405          debug!("adjustment={:?}", adjustment);
  406          match **adjustment {
  407              ty::AutoDerefRef(ty::AutoDerefRef {autoderefs, autoref: opt_autoref}) => {
  408                  let expr_ty = rcx.resolve_node_type(expr.id);
  409                  constrain_autoderefs(rcx, expr, autoderefs, expr_ty);
  410                  for autoref in opt_autoref.iter() {
  411                      link_autoref(rcx, expr, autoderefs, autoref);
  412  
  413                      // Require that the resulting region encompasses
  414                      // the current node.
  415                      //
  416                      // FIXME(#6268) remove to support nested method calls
  417                      constrain_regions_in_type_of_node(
  418                          rcx, expr.id, ty::ReScope(expr.id),
  419                          infer::AutoBorrow(expr.span));
  420                  }
  421              }
  422              ty::AutoObject(ast::BorrowedSigil, Some(trait_region), _, _, _, _) => {
  423                  // Determine if we are casting `expr` to an trait
  424                  // instance.  If so, we have to be sure that the type of
  425                  // the source obeys the trait's region bound.
  426                  //
  427                  // Note: there is a subtle point here concerning type
  428                  // parameters.  It is possible that the type of `source`
  429                  // contains type parameters, which in turn may contain
  430                  // regions that are not visible to us (only the caller
  431                  // knows about them).  The kind checker is ultimately
  432                  // responsible for guaranteeing region safety in that
  433                  // particular case.  There is an extensive comment on the
  434                  // function check_cast_for_escaping_regions() in kind.rs
  435                  // explaining how it goes about doing that.
  436  
  437                  let source_ty = rcx.fcx.expr_ty(expr);
  438                  constrain_regions_in_type(rcx, trait_region,
  439                                              infer::RelateObjectBound(expr.span), source_ty);
  440              }
  441              _ => {}
  442          }
  443      }
  444  
  445      match expr.node {
  446          ast::ExprCall(callee, ref args) => {
  447              constrain_callee(rcx, callee.id, expr, callee);
  448              constrain_call(rcx,
  449                             Some(callee.id),
  450                             expr,
  451                             None,
  452                             args.as_slice(),
  453                             false);
  454  
  455              visit::walk_expr(rcx, expr, ());
  456          }
  457  
  458          ast::ExprMethodCall(_, _, ref args) => {
  459              constrain_call(rcx, None, expr, Some(*args.get(0)),
  460                             args.slice_from(1), false);
  461  
  462              visit::walk_expr(rcx, expr, ());
  463          }
  464  
  465          ast::ExprAssign(lhs, _) => {
  466              adjust_borrow_kind_for_assignment_lhs(rcx, lhs);
  467              visit::walk_expr(rcx, expr, ());
  468          }
  469  
  470          ast::ExprAssignOp(_, lhs, rhs) => {
  471              if has_method_map {
  472                  constrain_call(rcx, None, expr, Some(lhs), [rhs], true);
  473              }
  474  
  475              adjust_borrow_kind_for_assignment_lhs(rcx, lhs);
  476  
  477              visit::walk_expr(rcx, expr, ());
  478          }

  481          ast::ExprBinary(_, lhs, rhs) if has_method_map => {
  482              // As `expr_method_call`, but the call is via an
  483              // overloaded op.  Note that we (sadly) currently use an
  484              // implicit "by ref" sort of passing style here.  This
  485              // should be converted to an adjustment!
  486              constrain_call(rcx, None, expr, Some(lhs), [rhs], true);
  487  
  488              visit::walk_expr(rcx, expr, ());
  489          }
  490  
  491          ast::ExprUnary(_, lhs) if has_method_map => {
  492              // As above.
  493              constrain_call(rcx, None, expr, Some(lhs), [], true);
  494  
  495              visit::walk_expr(rcx, expr, ());
  496          }
  497  
  498          ast::ExprUnary(ast::UnDeref, base) => {
  499              // For *a, the lifetime of a must enclose the deref
  500              let method_call = MethodCall::expr(expr.id);
  501              let base_ty = match rcx.fcx.inh.method_map.borrow().find(&method_call) {
  502                  Some(method) => {
  503                      constrain_call(rcx, None, expr, Some(base), [], true);
  504                      ty::ty_fn_ret(method.ty)
  505                  }
  506                  None => rcx.resolve_node_type(base.id)
  507              };
  508              match ty::get(base_ty).sty {
  509                  ty::ty_rptr(r_ptr, _) => {
  510                      mk_subregion_due_to_dereference(rcx, expr.span,
  511                                                      ty::ReScope(expr.id), r_ptr);
  512                  }
  513                  _ => {}
  514              }
  515  
  516              visit::walk_expr(rcx, expr, ());
  517          }
  518  
  519          ast::ExprIndex(vec_expr, _) => {
  520              // For a[b], the lifetime of a must enclose the deref
  521              let vec_type = rcx.resolve_expr_type_adjusted(vec_expr);
  522              constrain_index(rcx, expr, vec_type);
  523  
  524              visit::walk_expr(rcx, expr, ());
  525          }

  527          ast::ExprCast(source, _) => {
  528              // Determine if we are casting `source` to an trait

  605  fn check_expr_fn_block(rcx: &mut Rcx,
  606                         expr: &ast::Expr,
  607                         body: &ast::Block) {
  608      let tcx = rcx.fcx.tcx();
  609      let function_type = rcx.resolve_node_type(expr.id);
  610      match ty::get(function_type).sty {
  611          ty::ty_closure(~ty::ClosureTy {
  612                  sigil: ast::BorrowedSigil, region: region, ..}) => {
  613              let freevars = freevars::get_freevars(tcx, expr.id);
  614              if freevars.is_empty() {
  615                  // No free variables means that the environment
  616                  // will be NULL at runtime and hence the closure
  617                  // has static lifetime.
  618              } else {
  619                  // Closure must not outlive the variables it closes over.
  620                  constrain_free_variables(rcx, region, expr, freevars);
  621  
  622                  // Closure cannot outlive the appropriate temporary scope.
  623                  let s = rcx.repeating_scope;
  624                  rcx.fcx.mk_subr(true, infer::InfStackClosure(expr.span),
  625                                  region, ty::ReScope(s));
  626              }
  627          }
  628          _ => ()
  629      }
  630  
  631      let repeating_scope = rcx.set_repeating_scope(body.id);
  632      visit::walk_expr(rcx, expr, ());
  633      rcx.set_repeating_scope(repeating_scope);
  634  
  635      match ty::get(function_type).sty {
  636          ty::ty_closure(~ty::ClosureTy {sigil: ast::BorrowedSigil, ..}) => {
  637              let freevars = freevars::get_freevars(tcx, expr.id);
  638              propagate_upupvar_borrow_kind(rcx, expr, freevars);
  639          }
  640          _ => ()
  641      }

  643      fn constrain_free_variables(rcx: &mut Rcx,
  644                                  region: ty::Region,
  645                                  expr: &ast::Expr,
  646                                  freevars: freevars::freevar_info) {
  647          /*!
  648           * Make sure that all free variables referenced inside the closure
  649           * outlive the closure itself. Also, create an entry in the
  650           * upvar_borrows map with a region.
  651           */
  652  
  653          let tcx = rcx.fcx.ccx.tcx;
  654          let infcx = rcx.fcx.infcx();
  655          debug!("constrain_free_variables({}, {})",
  656                 region.repr(tcx), expr.repr(tcx));
  657          for freevar in freevars.iter() {
  658              debug!("freevar def is {:?}", freevar.def);
  659  
  660              // Identify the variable being closed over and its node-id.
  661              let def = freevar.def;
  662              let def_id = ast_util::def_id_of_def(def);
  663              assert!(def_id.krate == ast::LOCAL_CRATE);
  664              let upvar_id = ty::UpvarId { var_id: def_id.node,
  665                                           closure_expr_id: expr.id };
  666  
  667              // Create a region variable to represent this borrow. This borrow
  668              // must outlive the region on the closure.
  669              let origin = infer::UpvarRegion(upvar_id, expr.span);
  670              let freevar_region = infcx.next_region_var(origin);
  671              rcx.fcx.mk_subr(true, infer::FreeVariable(freevar.span, def_id.node),
  672                              region, freevar_region);
  673  
  674              // Create a UpvarBorrow entry. Note that we begin with a
  675              // const borrow_kind, but change it to either mut or
  676              // immutable as dictated by the uses.
  677              let upvar_borrow = ty::UpvarBorrow { kind: ty::ImmBorrow,
  678                                                   region: freevar_region };
  679              rcx.fcx.inh.upvar_borrow_map.borrow_mut().insert(upvar_id,
  680                                                               upvar_borrow);
  681  
  682              // Guarantee that the closure does not outlive the variable itself.
  683              let en_region = region_of_def(rcx.fcx, def);
  684              debug!("en_region = {}", en_region.repr(tcx));
  685              rcx.fcx.mk_subr(true, infer::FreeVariable(freevar.span, def_id.node),
  686                              region, en_region);
  687          }
  688      }

  690      fn propagate_upupvar_borrow_kind(rcx: &mut Rcx,
  691                                       expr: &ast::Expr,
  692                                       freevars: freevars::freevar_info) {
  693          let tcx = rcx.fcx.ccx.tcx;
  694          debug!("propagate_upupvar_borrow_kind({})", expr.repr(tcx));
  695          for freevar in freevars.iter() {
  696              // Because of the semi-hokey way that we are doing
  697              // borrow_kind inference, we need to check for
  698              // indirect dependencies, like so:
  699              //
  700              //     let mut x = 0;
  701              //     outer_call(|| {
  702              //         inner_call(|| {
  703              //             x = 1;
  704              //         });
  705              //     });
  706              //
  707              // Here, the `inner_call` is basically "reborrowing" the
  708              // outer pointer. With no other changes, `inner_call`
  709              // would infer that it requires a mutable borrow, but
  710              // `outer_call` would infer that a const borrow is
  711              // sufficient. This is because we haven't linked the
  712              // borrow_kind of the borrow that occurs in the inner
  713              // closure to the borrow_kind of the borrow in the outer
  714              // closure. Note that regions *are* naturally linked
  715              // because we have a proper inference scheme there.
  716              //
  717              // Anyway, for borrow_kind, we basically go back over now
  718              // after checking the inner closure (and hence
  719              // determining the final borrow_kind) and propagate that as
  720              // a constraint on the outer closure.
  721              match freevar.def {
  722                  ast::DefUpvar(var_id, _, outer_closure_id, _) => {
  723                      // thing being captured is itself an upvar:
  724                      let outer_upvar_id = ty::UpvarId {
  725                          var_id: var_id,
  726                          closure_expr_id: outer_closure_id };
  727                      let inner_upvar_id = ty::UpvarId {
  728                          var_id: var_id,
  729                          closure_expr_id: expr.id };
  730                      link_upvar_borrow_kind_for_nested_closures(rcx,
  731                                                                 inner_upvar_id,
  732                                                                 outer_upvar_id);
  733                  }
  734                  _ => {}
  735              }
  736          }
  737      }
  738  }

  962  fn constrain_regions_in_type(
  963      rcx: &mut Rcx,
  964      minimum_lifetime: ty::Region,
  965      origin: infer::SubregionOrigin,
  966      ty: ty::t) -> bool
  967  {
  968      /*!
  969       * Requires that any regions which appear in `ty` must be
  970       * superregions of `minimum_lifetime`.  Also enforces the constraint
  971       * that given a pointer type `&'r T`, T must not contain regions
  972       * that outlive 'r, as well as analogous constraints for other
  973       * lifetime'd types.
  974       *
  975       * This check prevents regions from being used outside of the block in
  976       * which they are valid.  Recall that regions represent blocks of
  977       * code or expressions: this requirement basically says "any place
  978       * that uses or may use a region R must be within the block of
  979       * code that R corresponds to."
  980       */
  981  
  982      let e = rcx.errors_reported;
  983      let tcx = rcx.fcx.ccx.tcx;
  984  
  985      debug!("constrain_regions_in_type(minimum_lifetime={}, ty={})",
  986             region_to_str(tcx, "", false, minimum_lifetime),
  987             ty_to_str(tcx, ty));
  988  
  989      relate_nested_regions(tcx, Some(minimum_lifetime), ty, |r_sub, r_sup| {
  990          debug!("relate_nested_regions(r_sub={}, r_sup={})",
  991                  r_sub.repr(tcx),
  992                  r_sup.repr(tcx));
  993  
  994          if r_sup.is_bound() || r_sub.is_bound() {
  995              // a bound region is one which appears inside an fn type.
  996              // (e.g., the `&` in `fn(&T)`).  Such regions need not be
  997              // constrained by `minimum_lifetime` as they are placeholders
  998              // for regions that are as-yet-unknown.
  999          } else if r_sub == minimum_lifetime {
 1000              rcx.fcx.mk_subr(
 1001                  true, origin,
 1002                  r_sub, r_sup);
 1003          } else {
 1004              rcx.fcx.mk_subr(
 1005                  true, infer::ReferenceOutlivesReferent(ty, origin.span()),
 1006                  r_sub, r_sup);
 1007          }
 1008      });
 1009  
 1010      return e == rcx.errors_reported;
 1011  }
