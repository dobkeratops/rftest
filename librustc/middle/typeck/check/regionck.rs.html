   11  /*!
   12  
   13  The region check is a final pass that runs over the AST after we have
   14  inferred the type constraints but before we have actually finalized
   15  the types.  Its purpose is to embed a variety of region constraints.
   16  Inserting these constraints as a separate pass is good because (1) it
   17  localizes the code that has to do with region inference and (2) often
   18  we cannot know what constraints are needed until the basic types have
   19  been inferred.
   20  
   21  ### Interaction with the borrow checker
   22  
   23  In general, the job of the borrowck module (which runs later) is to
   24  check that all soundness criteria are met, given a particular set of
   25  regions. The job of *this* module is to anticipate the needs of the
   26  borrow checker and infer regions that will satisfy its requirements.
   27  It is generally true that the inference doesn't need to be sound,
   28  meaning that if there is a bug and we inferred bad regions, the borrow
   29  checker should catch it. This is not entirely true though; for
   30  example, the borrow checker doesn't check subtyping, and it doesn't
   31  check that region pointers are always live when they are used. It
   32  might be worthwhile to fix this so that borrowck serves as a kind of
   33  verification step -- that would add confidence in the overall
   34  correctness of the compiler, at the cost of duplicating some type
   35  checks and effort.
   36  
   37  ### Inferring the duration of borrows, automatic and otherwise
   38  
   39  Whenever we introduce a borrowed pointer, for example as the result of
   40  a borrow expression `let x = &data`, the lifetime of the pointer `x`
   41  is always specified as a region inference variable. `regionck` has the
   42  job of adding constraints such that this inference variable is as
   43  narrow as possible while still accommodating all uses (that is, every
   44  dereference of the resulting pointer must be within the lifetime).
   45  
   46  #### Reborrows
   47  
   48  Generally speaking, `regionck` does NOT try to ensure that the data
   49  `data` will outlive the pointer `x`. That is the job of borrowck.  The
   50  one exception is when "re-borrowing" the contents of another borrowed
   51  pointer. For example, imagine you have a borrowed pointer `b` with
   52  lifetime L1 and you have an expression `&*b`. The result of this
   53  expression will be another borrowed pointer with lifetime L2 (which is
   54  an inference variable). The borrow checker is going to enforce the
   55  constraint that L2 < L1, because otherwise you are re-borrowing data
   56  for a lifetime larger than the original loan.  However, without the
   57  routines in this module, the region inferencer would not know of this
   58  dependency and thus it might infer the lifetime of L2 to be greater
   59  than L1 (issue #3148).
   60  
   61  There are a number of troublesome scenarios in the tests
   62  `region-dependent-*.rs`, but here is one example:
   63  
   64      struct Foo { i: int }
   65      struct Bar { foo: Foo  }
   66      fn get_i(x: &'a Bar) -> &'a int {
   67         let foo = &x.foo; // Lifetime L1
   68         &foo.i            // Lifetime L2
   69      }
   70  
   71  Note that this comes up either with `&` expressions, `ref`
   72  bindings, and `autorefs`, which are the three ways to introduce
   73  a borrow.
   74  
   75  The key point here is that when you are borrowing a value that
   76  is "guaranteed" by a borrowed pointer, you must link the
   77  lifetime of that borrowed pointer (L1, here) to the lifetime of
   78  the borrow itself (L2).  What do I mean by "guaranteed" by a
   79  borrowed pointer? I mean any data that is reached by first
   80  dereferencing a borrowed pointer and then either traversing
   81  interior offsets or owned pointers.  We say that the guarantor
   82  of such data it the region of the borrowed pointer that was
   83  traversed.  This is essentially the same as the ownership
   84  relation, except that a borrowed pointer never owns its
   85  contents.
   86  
   87  ### Inferring borrow kinds for upvars
   88  
   89  Whenever there is a closure expression, we need to determine how each
   90  upvar is used. We do this by initially assigning each upvar an
   91  immutable "borrow kind" (see `ty::BorrowKind` for details) and then
   92  "escalating" the kind as needed. The borrow kind proceeds according to
   93  the following lattice:
   94  
   95      ty::ImmBorrow -> ty::UniqueImmBorrow -> ty::MutBorrow
   96  
   97  So, for example, if we see an assignment `x = 5` to an upvar `x`, we
   98  will promote its borrow kind to mutable borrow. If we see an `&mut x`
   99  we'll do the same. Naturally, this applies not just to the upvar, but
  100  to everything owned by `x`, so the result is the same for something
  101  like `x.f = 5` and so on (presuming `x` is not a borrowed pointer to a
  102  struct). These adjustments are performed in
  103  `adjust_upvar_borrow_kind()` (you can trace backwards through the code
  104  from there).
  105  
  106  The fact that we are inferring borrow kinds as we go results in a
  107  semi-hacky interaction with mem-categorization. In particular,
  108  mem-categorization will query the current borrow kind as it
  109  categorizes, and we'll return the *current* value, but this may get
  110  adjusted later. Therefore, in this module, we generally ignore the
  111  borrow kind (and derived mutabilities) that are returned from
  112  mem-categorization, since they may be inaccurate. (Another option
  113  would be to use a unification scheme, where instead of returning a
  114  concrete borrow kind like `ty::ImmBorrow`, we return a
  115  `ty::InferBorrow(upvar_id)` or something like that, but this would
  116  then mean that all later passes would have to check for these figments
  117  and report an error, and it just seems like more mess in the end.)
  118  
  119  */

  164  fn region_of_def(fcx: &FnCtxt, def: ast::Def) -> ty::Region {
  165      /*!
  166       * Returns the validity region of `def` -- that is, how long
  167       * is `def` valid?
  168       */
  169  
  170      let tcx = fcx.tcx();
  171      match def {
  172          DefLocal(node_id, _) | DefArg(node_id, _) |
  173          DefBinding(node_id, _) => {
  174              tcx.region_maps.var_region(node_id)
  175          }
  176          DefUpvar(_, subdef, closure_id, body_id) => {
  177              match ty::ty_closure_store(fcx.node_ty(closure_id)) {
  178                  ty::RegionTraitStore(..) => region_of_def(fcx, *subdef),
  179                  ty::UniqTraitStore => ReScope(body_id)
  180              }
  181          }
  182          _ => {
  183              tcx.sess.bug(format!("unexpected def in region_of_def: {:?}",
  184                                def))
  185          }
  186      }
  187  }

  200      pub fn resolve_type(&self, unresolved_ty: ty::t) -> ty::t {
  201          /*!
  202           * Try to resolve the type for the given node, returning
  203           * t_err if an error results.  Note that we never care
  204           * about the details of the error, the same error will be
  205           * detected and reported in the writeback phase.
  206           *
  207           * Note one important point: we do not attempt to resolve
  208           * *region variables* here.  This is because regionck is
  209           * essentially adding constraints to those region variables
  210           * and so may yet influence how they are resolved.
  211           *
  212           * Consider this silly example:
  213           *
  214           *     fn borrow(x: &int) -> &int {x}
  215           *     fn foo(x: @int) -> int {  // block: B
  216           *         let b = borrow(x);    // region: <R0>
  217           *         *b
  218           *     }
  219           *
  220           * Here, the region of `b` will be `<R0>`.  `<R0>` is
  221           * constrainted to be some subregion of the block B and some
  222           * superregion of the call.  If we forced it now, we'd choose
  223           * the smaller region (the call).  But that would make the *b
  224           * illegal.  Since we don't resolve, the type of b will be
  225           * `&<R0>.int` and then `*b` will require that `<R0>` be
  226           * bigger than the let and the `*b` expression, so we will
  227           * effectively resolve `<R0>` to be the block B.
  228           */
  229          match resolve_type(self.fcx.infcx(), unresolved_ty,
  230                             resolve_and_force_all_but_regions) {
  231              Ok(t) => t,
  232              Err(_) => ty::mk_err()
  233          }
  234      }

  362      let tcx = rcx.fcx.tcx();
  363      debug!("regionck::visit_pat(pat={})", pat.repr(tcx));
  364      pat_util::pat_bindings(tcx.def_map, pat, |_, id, span, _| {
  365          // If we have a variable that contains region'd data, that
  366          // data will be accessible from anywhere that the variable is
  367          // accessed. We must be wary of loops like this:
  368          //
  369          //     // from src/test/compile-fail/borrowck-lend-flow.rs
  370          //     let mut v = ~3, w = ~4;
  371          //     let mut x = &mut w;
  372          //     loop {
  373          //         **x += 1;   // (2)
  374          //         borrow(v);  //~ ERROR cannot borrow
  375          //         x = &mut v; // (1)
  376          //     }
  377          //
  378          // Typically, we try to determine the region of a borrow from
  379          // those points where it is dereferenced. In this case, one
  380          // might imagine that the lifetime of `x` need only be the
  381          // body of the loop. But of course this is incorrect because
  382          // the pointer that is created at point (1) is consumed at
  383          // point (2), meaning that it must be live across the loop
  384          // iteration. The easiest way to guarantee this is to require
  385          // that the lifetime of any regions that appear in a
  386          // variable's type enclose at least the variable's scope.
  387  
  388          let var_region = tcx.region_maps.var_region(id);
  389          constrain_regions_in_type_of_node(
  390              rcx, id, var_region,
  391              infer::BindingTypeIsNotValidAtDecl(span));
  392      })
  393  }

  395  fn visit_expr(rcx: &mut Rcx, expr: &ast::Expr) {
  396      debug!("regionck::visit_expr(e={}, repeating_scope={:?})",
  397             expr.repr(rcx.fcx.tcx()), rcx.repeating_scope);
  398  
  399      let method_call = MethodCall::expr(expr.id);
  400      let has_method_map = rcx.fcx.inh.method_map.borrow().contains_key(&method_call);
  401  
  402      // Check any autoderefs or autorefs that appear.
  403      for &adjustment in rcx.fcx.inh.adjustments.borrow().find(&expr.id).iter() {
  404          debug!("adjustment={:?}", adjustment);
  405          match **adjustment {
  406              ty::AutoDerefRef(ty::AutoDerefRef {autoderefs, autoref: opt_autoref}) => {
  407                  let expr_ty = rcx.resolve_node_type(expr.id);
  408                  constrain_autoderefs(rcx, expr, autoderefs, expr_ty);
  409                  for autoref in opt_autoref.iter() {
  410                      link_autoref(rcx, expr, autoderefs, autoref);
  411  
  412                      // Require that the resulting region encompasses
  413                      // the current node.
  414                      //
  415                      // FIXME(#6268) remove to support nested method calls
  416                      constrain_regions_in_type_of_node(
  417                          rcx, expr.id, ty::ReScope(expr.id),
  418                          infer::AutoBorrow(expr.span));
  419                  }
  420              }
  421              ty::AutoObject(ty::RegionTraitStore(trait_region, _), _, _, _) => {
  422                  // Determine if we are casting `expr` to an trait
  423                  // instance.  If so, we have to be sure that the type of
  424                  // the source obeys the trait's region bound.
  425                  //
  426                  // Note: there is a subtle point here concerning type
  427                  // parameters.  It is possible that the type of `source`
  428                  // contains type parameters, which in turn may contain
  429                  // regions that are not visible to us (only the caller
  430                  // knows about them).  The kind checker is ultimately
  431                  // responsible for guaranteeing region safety in that
  432                  // particular case.  There is an extensive comment on the
  433                  // function check_cast_for_escaping_regions() in kind.rs
  434                  // explaining how it goes about doing that.
  435  
  436                  let source_ty = rcx.fcx.expr_ty(expr);
  437                  constrain_regions_in_type(rcx, trait_region,
  438                                              infer::RelateObjectBound(expr.span), source_ty);
  439              }
  440              _ => {}
  441          }
  442      }
  443  
  444      match expr.node {
  445          ast::ExprCall(callee, ref args) => {
  446              constrain_callee(rcx, callee.id, expr, callee);
  447              constrain_call(rcx,
  448                             Some(callee.id),
  449                             expr,
  450                             None,
  451                             args.as_slice(),
  452                             false);
  453

  466              visit::walk_expr(rcx, expr, ());
  467          }
  468  
  469          ast::ExprAssignOp(_, lhs, rhs) => {

  482              // overloaded op.  Note that we (sadly) currently use an
  483              // implicit "by ref" sort of passing style here.  This
  484              // should be converted to an adjustment!
  485              constrain_call(rcx, None, expr, Some(lhs), [rhs], true);

  489

  492              constrain_call(rcx, None, expr, Some(lhs), [], true);

  502                      constrain_call(rcx, None, expr, Some(base), [], true);
  503                      ty::ty_fn_ret(method.ty)
  504                  }
  505                  None => rcx.resolve_node_type(base.id)
  506              };
  507              match ty::get(base_ty).sty {
  508                  ty::ty_rptr(r_ptr, _) => {
  509                      mk_subregion_due_to_dereference(rcx, expr.span,
  510                                                      ty::ReScope(expr.id), r_ptr);
  511                  }

  513              }

  515              visit::walk_expr(rcx, expr, ());

  606  fn check_expr_fn_block(rcx: &mut Rcx,
  607                         expr: &ast::Expr,
  608                         body: &ast::Block) {
  609      let tcx = rcx.fcx.tcx();
  610      let function_type = rcx.resolve_node_type(expr.id);
  611      match ty::get(function_type).sty {
  612          ty::ty_closure(~ty::ClosureTy {
  613                  store: ty::RegionTraitStore(region, _), ..}) => {
  614              let freevars = freevars::get_freevars(tcx, expr.id);
  615              if freevars.is_empty() {
  616                  // No free variables means that the environment
  617                  // will be NULL at runtime and hence the closure
  618                  // has static lifetime.
  619              } else {
  620                  // Closure must not outlive the variables it closes over.
  621                  constrain_free_variables(rcx, region, expr, freevars);
  622  
  623                  // Closure cannot outlive the appropriate temporary scope.
  624                  let s = rcx.repeating_scope;
  625                  rcx.fcx.mk_subr(true, infer::InfStackClosure(expr.span),
  626                                  region, ty::ReScope(s));
  627              }
  628          }
  629          _ => ()
  630      }
  631  
  632      let repeating_scope = rcx.set_repeating_scope(body.id);
  633      visit::walk_expr(rcx, expr, ());
  634      rcx.set_repeating_scope(repeating_scope);
  635  
  636      match ty::get(function_type).sty {
  637          ty::ty_closure(~ty::ClosureTy {store: ty::RegionTraitStore(..), ..}) => {
  638              let freevars = freevars::get_freevars(tcx, expr.id);
  639              propagate_upupvar_borrow_kind(rcx, expr, freevars);
  640          }
  641          _ => ()
  642      }

  644      fn constrain_free_variables(rcx: &mut Rcx,
  645                                  region: ty::Region,
  646                                  expr: &ast::Expr,
  647                                  freevars: freevars::freevar_info) {
  648          /*!
  649           * Make sure that all free variables referenced inside the closure
  650           * outlive the closure itself. Also, create an entry in the
  651           * upvar_borrows map with a region.
  652           */
  653  
  654          let tcx = rcx.fcx.ccx.tcx;
  655          let infcx = rcx.fcx.infcx();
  656          debug!("constrain_free_variables({}, {})",
  657                 region.repr(tcx), expr.repr(tcx));
  658          for freevar in freevars.iter() {
  659              debug!("freevar def is {:?}", freevar.def);
  660  
  661              // Identify the variable being closed over and its node-id.
  662              let def = freevar.def;
  663              let def_id = ast_util::def_id_of_def(def);
  664              assert!(def_id.krate == ast::LOCAL_CRATE);
  665              let upvar_id = ty::UpvarId { var_id: def_id.node,
  666                                           closure_expr_id: expr.id };
  667  
  668              // Create a region variable to represent this borrow. This borrow
  669              // must outlive the region on the closure.
  670              let origin = infer::UpvarRegion(upvar_id, expr.span);
  671              let freevar_region = infcx.next_region_var(origin);
  672              rcx.fcx.mk_subr(true, infer::FreeVariable(freevar.span, def_id.node),
  673                              region, freevar_region);
  674  
  675              // Create a UpvarBorrow entry. Note that we begin with a
  676              // const borrow_kind, but change it to either mut or
  677              // immutable as dictated by the uses.
  678              let upvar_borrow = ty::UpvarBorrow { kind: ty::ImmBorrow,
  679                                                   region: freevar_region };
  680              rcx.fcx.inh.upvar_borrow_map.borrow_mut().insert(upvar_id,
  681                                                               upvar_borrow);
  682  
  683              // Guarantee that the closure does not outlive the variable itself.
  684              let en_region = region_of_def(rcx.fcx, def);
  685              debug!("en_region = {}", en_region.repr(tcx));
  686              rcx.fcx.mk_subr(true, infer::FreeVariable(freevar.span, def_id.node),
  687                              region, en_region);
  688          }
  689      }

  691      fn propagate_upupvar_borrow_kind(rcx: &mut Rcx,
  692                                       expr: &ast::Expr,
  693                                       freevars: freevars::freevar_info) {
  694          let tcx = rcx.fcx.ccx.tcx;
  695          debug!("propagate_upupvar_borrow_kind({})", expr.repr(tcx));
  696          for freevar in freevars.iter() {
  697              // Because of the semi-hokey way that we are doing
  698              // borrow_kind inference, we need to check for
  699              // indirect dependencies, like so:
  700              //
  701              //     let mut x = 0;
  702              //     outer_call(|| {
  703              //         inner_call(|| {
  704              //             x = 1;
  705              //         });
  706              //     });
  707              //
  708              // Here, the `inner_call` is basically "reborrowing" the
  709              // outer pointer. With no other changes, `inner_call`
  710              // would infer that it requires a mutable borrow, but
  711              // `outer_call` would infer that a const borrow is
  712              // sufficient. This is because we haven't linked the
  713              // borrow_kind of the borrow that occurs in the inner
  714              // closure to the borrow_kind of the borrow in the outer
  715              // closure. Note that regions *are* naturally linked
  716              // because we have a proper inference scheme there.
  717              //
  718              // Anyway, for borrow_kind, we basically go back over now
  719              // after checking the inner closure (and hence
  720              // determining the final borrow_kind) and propagate that as
  721              // a constraint on the outer closure.
  722              match freevar.def {
  723                  ast::DefUpvar(var_id, _, outer_closure_id, _) => {
  724                      // thing being captured is itself an upvar:
  725                      let outer_upvar_id = ty::UpvarId {
  726                          var_id: var_id,
  727                          closure_expr_id: outer_closure_id };
  728                      let inner_upvar_id = ty::UpvarId {
  729                          var_id: var_id,
  730                          closure_expr_id: expr.id };
  731                      link_upvar_borrow_kind_for_nested_closures(rcx,
  732                                                                 inner_upvar_id,
  733                                                                 outer_upvar_id);
  734                  }
  735                  _ => {}
  736              }
  737          }
  738      }
  739  }

  769  fn constrain_call(rcx: &mut Rcx,
  770                    // might be expr_call, expr_method_call, or an overloaded
  771                    // operator
  772                    fn_expr_id: Option<ast::NodeId>,
  773                    call_expr: &ast::Expr,
  774                    receiver: Option<@ast::Expr>,
  775                    arg_exprs: &[@ast::Expr],
  776                    implicitly_ref_args: bool) {
  777      //! Invoked on every call site (i.e., normal calls, method calls,
  778      //! and overloaded operators). Constrains the regions which appear
  779      //! in the type of the function. Also constrains the regions that
  780      //! appear in the arguments appropriately.
  781  
  782      let tcx = rcx.fcx.tcx();
  783      debug!("constrain_call(call_expr={}, \
  784              receiver={}, \
  785              arg_exprs={}, \
  786              implicitly_ref_args={:?})",
  787              call_expr.repr(tcx),
  788              receiver.repr(tcx),
  789              arg_exprs.repr(tcx),
  790              implicitly_ref_args);
  791      let callee_ty = match fn_expr_id {
  792          Some(id) => rcx.resolve_node_type(id),
  793          None => rcx.resolve_method_type(MethodCall::expr(call_expr.id))
  794                     .expect("call should have been to a method")
  795      };
  796      if ty::type_is_error(callee_ty) {
  797          // Bail, as function type is unknown
  798          return;
  799      }
  800      let fn_sig = ty::ty_fn_sig(callee_ty);
  801  
  802      // `callee_region` is the scope representing the time in which the
  803      // call occurs.
  804      //
  805      // FIXME(#6268) to support nested method calls, should be callee_id
  806      let callee_scope = call_expr.id;

  808  
  809      for &arg_expr in arg_exprs.iter() {
  810          debug!("Argument");
  811  
  812          // ensure that any regions appearing in the argument type are
  813          // valid for at least the lifetime of the function:
  814          constrain_regions_in_type_of_node(
  815              rcx, arg_expr.id, callee_region,
  816              infer::CallArg(arg_expr.span));
  817  
  818          // unfortunately, there are two means of taking implicit
  819          // references, and we need to propagate constraints as a
  820          // result. modes are going away and the "DerefArgs" code
  821          // should be ported to use adjustments
  822          if implicitly_ref_args {
  823              link_by_ref(rcx, arg_expr, callee_scope);
  824          }
  825      }
  826  
  827      // as loop above, but for receiver
  828      for &r in receiver.iter() {
  829          debug!("Receiver");
  830          constrain_regions_in_type_of_node(
  831              rcx, r.id, callee_region, infer::CallRcvr(r.span));
  832          if implicitly_ref_args {
  833              link_by_ref(rcx, r, callee_scope);
  834          }
  835      }
  836  
  837      // constrain regions that may appear in the return type to be
  838      // valid for the function call:
  839      constrain_regions_in_type(
  840          rcx, callee_region, infer::CallReturn(call_expr.span),
  841          fn_sig.output);
  842  }

  844  fn constrain_autoderefs(rcx: &mut Rcx,
  845                          deref_expr: &ast::Expr,
  846                          derefs: uint,
  847                          mut derefd_ty: ty::t) {
  848      /*!
  849       * Invoked on any auto-dereference that occurs.  Checks that if
  850       * this is a region pointer being dereferenced, the lifetime of
  851       * the pointer includes the deref expr.

  857                 i, derefs);

  865                  let self_ty = *fn_sig.inputs.get(0);

  867                      ty::ty_rptr(r, ref m) => (m.mutbl, r),
  868                      _ => rcx.tcx().sess.span_bug(deref_expr.span,

  875                      link_region(mc.typer, deref_expr.span, r, m, self_cmt);
  876                  }
  877  
  878                  // Specialized version of constrain_call.
  879                  constrain_regions_in_type(rcx, r_deref_expr,
  880                                            infer::CallRcvr(deref_expr.span),
  881                                            self_ty);
  882                  constrain_regions_in_type(rcx, r_deref_expr,
  883                                            infer::CallReturn(deref_expr.span),
  884                                            fn_sig.output);
  885                  fn_sig.output
  886              }
  887              None => derefd_ty
  888          };

  891              ty::ty_rptr(r_ptr, _) => {

  900              /* if this type can't be dereferenced, then there's already an error
  901                 in the session saying so. Just bail out for now */
  902              None => break
  903          }
  904      }

  945      origin: infer::SubregionOrigin) -> bool

  967  fn constrain_regions_in_type(
  968      rcx: &mut Rcx,
  969      minimum_lifetime: ty::Region,
  970      origin: infer::SubregionOrigin,
  971      ty: ty::t) -> bool
  972  {
  973      /*!
  974       * Requires that any regions which appear in `ty` must be
  975       * superregions of `minimum_lifetime`.  Also enforces the constraint
  976       * that given a pointer type `&'r T`, T must not contain regions
  977       * that outlive 'r, as well as analogous constraints for other
  978       * lifetime'd types.
  979       *
  980       * This check prevents regions from being used outside of the block in
  981       * which they are valid.  Recall that regions represent blocks of
  982       * code or expressions: this requirement basically says "any place
  983       * that uses or may use a region R must be within the block of
  984       * code that R corresponds to."
  985       */
  986  
  987      let e = rcx.errors_reported;
  988      let tcx = rcx.fcx.ccx.tcx;
  989  
  990      debug!("constrain_regions_in_type(minimum_lifetime={}, ty={})",
  991             region_to_str(tcx, "", false, minimum_lifetime),
  992             ty_to_str(tcx, ty));
  993  
  994      relate_nested_regions(tcx, Some(minimum_lifetime), ty, |r_sub, r_sup| {
  995          debug!("relate_nested_regions(r_sub={}, r_sup={})",
  996                  r_sub.repr(tcx),
  997                  r_sup.repr(tcx));
  998  
  999          if r_sup.is_bound() || r_sub.is_bound() {
 1000              // a bound region is one which appears inside an fn type.
 1001              // (e.g., the `&` in `fn(&T)`).  Such regions need not be
 1002              // constrained by `minimum_lifetime` as they are placeholders
 1003              // for regions that are as-yet-unknown.
 1004          } else if r_sub == minimum_lifetime {
 1005              rcx.fcx.mk_subr(
 1006                  true, origin,
 1007                  r_sub, r_sup);
 1008          } else {
 1009              rcx.fcx.mk_subr(
 1010                  true, infer::ReferenceOutlivesReferent(ty, origin.span()),
 1011                  r_sub, r_sup);
 1012          }
 1013      });
 1014  
 1015      return e == rcx.errors_reported;
 1016  }
