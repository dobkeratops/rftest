   13  # Translation of expressions.
   14  
   15  ## Recommended entry point
   16  
   17  If you wish to translate an expression, the preferred way to do
   18  so is to use:
   19  
   20      expr::trans_into(block, expr, Dest) -> block
   21  
   22  This will generate code that evaluates `expr`, storing the result into
   23  `Dest`, which must either be the special flag ignore (throw the result
   24  away) or be a pointer to memory of the same type/size as the
   25  expression.  It returns the resulting basic block.  This form will
   26  handle all automatic adjustments for you. The value will be moved if
   27  its type is linear and copied otherwise.
   28  
   29  ## Translation to a datum
   30  
   31  In some cases, `trans_into()` is too narrow of an interface.
   32  Generally this occurs either when you know that the result value is
   33  going to be a scalar, or when you need to evaluate the expression into
   34  some memory location so you can go and inspect it (e.g., assignments,
   35  `match` expressions, the `&` operator).
   36  
   37  In such cases, you want the following function:
   38  
   39      trans_to_datum(block, expr) -> DatumBlock
   40  
   41  This function generates code to evaluate the expression and return a
   42  `Datum` describing where the result is to be found.  This function
   43  tries to return its result in the most efficient way possible, without
   44  introducing extra copies or sacrificing information.  Therefore, for
   45  lvalue expressions, you always get a by-ref `Datum` in return that
   46  points at the memory for this lvalue.  For rvalue expressions, we will
   47  return a by-value `Datum` whenever possible, but it is often necessary
   48  to allocate a stack slot, store the result of the rvalue in there, and
   49  then return a pointer to the slot (see the discussion later on about
   50  the different kinds of rvalues).
   51  
   52  NB: The `trans_to_datum()` function does perform adjustments, but
   53  since it returns a pointer to the value "in place" it does not handle
   54  moves.  If you wish to copy/move the value returned into a new
   55  location, you should use the Datum method `store_to()` (move or copy
   56  depending on type). You can also use `move_to()` (force move) or
   57  `copy_to()` (force copy) for special situations.
   58  
   59  ## Translating local variables
   60  
   61  `trans_local_var()` can be used to trans a ref to a local variable
   62  that is not an expression.  This is needed for captures.
   63  
   64  ## Ownership and cleanups
   65  
   66  The current system for cleanups associates required cleanups with
   67  block contexts.  Block contexts are structured into a tree that
   68  resembles the code itself.  Not every block context has cleanups
   69  associated with it, only those blocks that have a kind of
   70  `block_scope`.  See `common::block_kind` for more details.
   71  
   72  If you invoke `trans_into()`, no cleanup is scheduled for you.  The
   73  value is written into the given destination and is assumed to be owned
   74  by that destination.
   75  
   76  When you invoke `trans_to_datum()` on an rvalue, the resulting
   77  datum/value will have an appropriate cleanup scheduled for the
   78  innermost cleanup scope.  If you later use `move_to()` or
   79  `drop_val()`, this cleanup will be canceled.
   80  
   81  During the evaluation of an expression, temporary cleanups are created
   82  and later canceled.  These represent intermediate or partial results
   83  which must be cleaned up in the event of task failure.
   84  
   85  ## Implementation details
   86  
   87  We divide expressions into three categories, based on how they are most
   88  naturally implemented:
   89  
   90  1. Lvalues
   91  2. Datum rvalues
   92  3. DPS rvalues
   93  4. Statement rvalues
   94  
   95  Lvalues always refer to user-assignable memory locations.
   96  Translating those always results in a by-ref datum; this introduces
   97  no inefficiencies into the generated code, because all lvalues are
   98  naturally addressable.
   99  
  100  Datum rvalues are rvalues that always generate datums as a result.
  101  These are generally scalar results, such as `a+b` where `a` and `b`
  102  are integers.
  103  
  104  DPS rvalues are rvalues that, when translated, must be given a
  105  memory location to write into (or the Ignore flag).  These are
  106  generally expressions that produce structural results that are
  107  larger than one word (e.g., a struct literal), but also expressions
  108  (like `if`) that involve control flow (otherwise we'd have to
  109  generate phi nodes).
  110  
  111  Finally, statement rvalues are rvalues that always produce a nil
  112  return type, such as `while` loops or assignments (`a = b`).
  113  
  114  */

  184  pub fn trans_to_datum(bcx: @mut Block, expr: @ast::Expr) -> DatumBlock {
  185      debug!("trans_to_datum(expr=%s)", bcx.expr_to_str(expr));
  186  
  187      let mut bcx = bcx;
  188      let mut datum = unpack_datum!(bcx, trans_to_datum_unadjusted(bcx, expr));
  189      let adjustment = match bcx.tcx().adjustments.find_copy(&expr.id) {
  190          None => { return DatumBlock {bcx: bcx, datum: datum}; }
  191          Some(adj) => { adj }
  192      };
  193      debug!("unadjusted datum: %s", datum.to_str(bcx.ccx()));
  194      match *adjustment {
  195          AutoAddEnv(*) => {
  196              datum = unpack_datum!(bcx, add_env(bcx, expr, datum));
  197          }
  198          AutoDerefRef(ref adj) => {
  199              if adj.autoderefs > 0 {
  200                  datum =
  201                      unpack_datum!(
  202                          bcx,
  203                          datum.autoderef(bcx, expr.span,
  204                                          expr.id, adj.autoderefs));
  205              }
  206  
  207              datum = match adj.autoref {
  208                  None => {
  209                      datum
  210                  }
  211                  Some(AutoUnsafe(*)) | // region + unsafe ptrs have same repr
  212                  Some(AutoPtr(*)) => {
  213                      unpack_datum!(bcx, auto_ref(bcx, datum))
  214                  }
  215                  Some(AutoBorrowVec(*)) => {
  216                      unpack_datum!(bcx, auto_slice(bcx, adj.autoderefs,
  217                                                    expr, datum))
  218                  }
  219                  Some(AutoBorrowVecRef(*)) => {
  220                      unpack_datum!(bcx, auto_slice_and_ref(bcx, adj.autoderefs,
  221                                                            expr, datum))
  222                  }
  223                  Some(AutoBorrowFn(*)) => {
  224                      let adjusted_ty = ty::adjust_ty(bcx.tcx(), expr.span,
  225                                                      datum.ty, Some(adjustment));
  226                      unpack_datum!(bcx, auto_borrow_fn(bcx, adjusted_ty, datum))
  227                  }
  228                  Some(AutoBorrowObj(*)) => {
  229                      unpack_datum!(bcx, auto_borrow_obj(
  230                          bcx, adj.autoderefs, expr, datum))
  231                  }
  232              };
  233          }
  234      }
  235      debug!("after adjustments, datum=%s", datum.to_str(bcx.ccx()));
  236      return DatumBlock {bcx: bcx, datum: datum};

  253      fn auto_slice(bcx: @mut Block,
  254                    autoderefs: uint,
  255                    expr: &ast::Expr,
  256                    datum: Datum) -> DatumBlock {
  257          // This is not the most efficient thing possible; since slices
  258          // are two words it'd be better if this were compiled in
  259          // 'dest' mode, but I can't find a nice way to structure the
  260          // code and keep it DRY that accommodates that use case at the
  261          // moment.
  262  
  263          let tcx = bcx.tcx();
  264          let unit_ty = ty::sequence_element_type(tcx, datum.ty);
  265  
  266          let (bcx, base, len) =
  267              datum.get_vec_base_and_len(bcx, expr.span, expr.id, autoderefs+1);
  268  
  269          // this type may have a different region/mutability than the
  270          // real one, but it will have the same runtime representation
  271          let slice_ty = ty::mk_evec(tcx,
  272                                     ty::mt { ty: unit_ty, mutbl: ast::MutImmutable },
  273                                     ty::vstore_slice(ty::re_static));
  274  
  275          let scratch = scratch_datum(bcx, slice_ty, "__adjust", false);
  276          Store(bcx, base, GEPi(bcx, scratch.val, [0u, abi::slice_elt_base]));
  277          Store(bcx, len, GEPi(bcx, scratch.val, [0u, abi::slice_elt_len]));
  278          DatumBlock {bcx: bcx, datum: scratch}
  279      }

  281      fn add_env(bcx: @mut Block, expr: &ast::Expr, datum: Datum) -> DatumBlock {
  282          // This is not the most efficient thing possible; since closures
  283          // are two words it'd be better if this were compiled in
  284          // 'dest' mode, but I can't find a nice way to structure the
  285          // code and keep it DRY that accommodates that use case at the
  286          // moment.
  287  
  288          let tcx = bcx.tcx();
  289          let closure_ty = expr_ty_adjusted(bcx, expr);
  290          debug!("add_env(closure_ty=%s)", closure_ty.repr(tcx));
  291          let scratch = scratch_datum(bcx, closure_ty, "__adjust", false);
  292          let llfn = GEPi(bcx, scratch.val, [0u, abi::fn_field_code]);
  293          assert_eq!(datum.appropriate_mode(tcx), ByValue);
  294          Store(bcx, datum.to_appropriate_llval(bcx), llfn);
  295          let llenv = GEPi(bcx, scratch.val, [0u, abi::fn_field_box]);
  296          Store(bcx, base::null_env_ptr(bcx.ccx()), llenv);
  297          DatumBlock {bcx: bcx, datum: scratch}
  298      }

  308      fn auto_borrow_obj(mut bcx: @mut Block,
  309                         autoderefs: uint,
  310                         expr: @ast::Expr,
  311                         source_datum: Datum) -> DatumBlock {
  312          let tcx = bcx.tcx();
  313          let target_obj_ty = expr_ty_adjusted(bcx, expr);
  314          debug!("auto_borrow_obj(target=%s)",
  315                 target_obj_ty.repr(tcx));
  316  
  317          // Extract source store information
  318          let (source_store, source_mutbl) = match ty::get(source_datum.ty).sty {
  319              ty::ty_trait(_, _, s, m, _) => (s, m),
  320              _ => {
  321                  bcx.sess().span_bug(
  322                      expr.span,
  323                      fmt!("auto_borrow_trait_obj expected a trait, found %s",
  324                           source_datum.ty.repr(bcx.tcx())));
  325              }
  326          };
  327  
  328          // check if any borrowing is really needed or we could reuse the source_datum instead
  329          match ty::get(target_obj_ty).sty {
  330              ty::ty_trait(_, _, ty::RegionTraitStore(target_scope), target_mutbl, _) => {
  331                  if target_mutbl == ast::MutImmutable && target_mutbl == source_mutbl {
  332                      match source_store {
  333                          ty::RegionTraitStore(source_scope) => {
  334                              if tcx.region_maps.is_subregion_of(target_scope, source_scope) {
  335                                  return DatumBlock { bcx: bcx, datum: source_datum };
  336                              }
  337                          },
  338                          _ => {}
  339  
  340                      };
  341                  }
  342              },
  343              _ => {}
  344          }
  345  
  346          let scratch = scratch_datum(bcx, target_obj_ty,
  347                                      "__auto_borrow_obj", false);
  348  
  349          // Convert a @Object, ~Object, or &Object pair into an &Object pair.
  350  
  351          // Get a pointer to the source object, which is represented as
  352          // a (vtable, data) pair.
  353          let source_llval = source_datum.to_ref_llval(bcx);
  354  
  355          // Set the vtable field of the new pair
  356          let vtable_ptr = GEPi(bcx, source_llval, [0u, abi::trt_field_vtable]);
  357          let vtable = Load(bcx, vtable_ptr);
  358          Store(bcx, vtable, GEPi(bcx, scratch.val, [0u, abi::trt_field_vtable]));
  359  
  360          // Load the data for the source, which is either an @T,
  361          // ~T, or &T, depending on source_obj_ty.
  362          let source_data_ptr = GEPi(bcx, source_llval, [0u, abi::trt_field_box]);
  363          let source_data = Load(bcx, source_data_ptr); // always a ptr
  364          let target_data = match source_store {
  365              ty::BoxTraitStore(*) => {
  366                  // For deref of @T or @mut T, create a dummy datum and
  367                  // use the datum's deref method. This is more work
  368                  // than just calling GEPi ourselves, but it ensures
  369                  // that any write guards will be appropriate
  370                  // processed.  Note that we don't know the type T, so
  371                  // just substitute `i8`-- it doesn't really matter for
  372                  // our purposes right now.
  373                  let source_ty =
  374                      ty::mk_box(tcx,
  375                                 ty::mt {
  376                                     ty: ty::mk_i8(),
  377                                     mutbl: source_mutbl});
  378                  let source_datum =
  379                      Datum {val: source_data,
  380                             ty: source_ty,
  381                             mode: ByValue};
  382                  let derefd_datum =
  383                      unpack_datum!(bcx,
  384                                    source_datum.deref(bcx,
  385                                                       expr,
  386                                                       autoderefs));
  387                  derefd_datum.to_rptr(bcx).to_value_llval(bcx)
  388              }
  389              ty::UniqTraitStore(*) => {
  390                  // For a ~T box, there may or may not be a header,
  391                  // depending on whether the type T references managed
  392                  // boxes. However, since we do not *know* the type T
  393                  // for objects, this presents a hurdle. Our solution is
  394                  // to load the "borrow offset" from the type descriptor;
  395                  // this value will either be 0 or sizeof(BoxHeader), depending
  396                  // on the type T.
  397                  let llopaque =
  398                      PointerCast(bcx, source_data, Type::opaque().ptr_to());
  399                  let lltydesc_ptr_ptr =
  400                      PointerCast(bcx, vtable,
  401                                  bcx.ccx().tydesc_type.ptr_to().ptr_to());
  402                  let lltydesc_ptr =
  403                      Load(bcx, lltydesc_ptr_ptr);
  404                  let borrow_offset_ptr =
  405                      GEPi(bcx, lltydesc_ptr,
  406                           [0, abi::tydesc_field_borrow_offset]);
  407                  let borrow_offset =
  408                      Load(bcx, borrow_offset_ptr);
  409                  InBoundsGEP(bcx, llopaque, [borrow_offset])
  410              }
  411              ty::RegionTraitStore(*) => {
  412                  source_data
  413              }
  414          };
  415          Store(bcx, target_data,
  416                GEPi(bcx, scratch.val, [0u, abi::trt_field_box]));
  417  
  418          DatumBlock { bcx: bcx, datum: scratch }
  419      }
  420  }

  422  pub fn trans_into(bcx: @mut Block, expr: @ast::Expr, dest: Dest) -> @mut Block {
  423      if bcx.tcx().adjustments.contains_key(&expr.id) {
  424          // use trans_to_datum, which is mildly less efficient but
  425          // which will perform the adjustments:
  426          let datumblock = trans_to_datum(bcx, expr);
  427          return match dest {
  428              Ignore => datumblock.bcx,
  429              SaveIn(lldest) => datumblock.store_to(INIT, lldest)
  430          };
  431      }
  432  
  433      let ty = expr_ty(bcx, expr);
  434  
  435      debug!("trans_into_unadjusted(expr=%s, dest=%s)",
  436             bcx.expr_to_str(expr),
  437             dest.to_str(bcx.ccx()));
  438      let _indenter = indenter();
  439  
  440      debuginfo::set_source_location(bcx.fcx, expr.id, expr.span);
  441  
  442      let dest = {
  443          if ty::type_is_voidish(ty) {
  444              Ignore
  445          } else {
  446              dest
  447          }
  448      };
  449  
  450      let kind = bcx.expr_kind(expr);
  451      debug!("expr kind = %?", kind);
  452      return match kind {
  453          ty::LvalueExpr => {
  454              let datumblock = trans_lvalue_unadjusted(bcx, expr);
  455              match dest {
  456                  Ignore => datumblock.bcx,
  457                  SaveIn(lldest) => datumblock.store_to(INIT, lldest)
  458              }
  459          }
  460          ty::RvalueDatumExpr => {
  461              let datumblock = trans_rvalue_datum_unadjusted(bcx, expr);
  462              match dest {
  463                  Ignore => datumblock.drop_val(),
  464  
  465                  // When processing an rvalue, the value will be newly
  466                  // allocated, so we always `move_to` so as not to
  467                  // unnecessarily inc ref counts and so forth:
  468                  SaveIn(lldest) => datumblock.move_to(INIT, lldest)
  469              }
  470          }
  471          ty::RvalueDpsExpr => {
  472              trans_rvalue_dps_unadjusted(bcx, expr, dest)
  473          }
  474          ty::RvalueStmtExpr => {
  475              trans_rvalue_stmt_unadjusted(bcx, expr)
  476          }
  477      };
  478  }

  480  fn trans_lvalue(bcx: @mut Block, expr: @ast::Expr) -> DatumBlock {
  481      /*!
  482       *
  483       * Translates an lvalue expression, always yielding a by-ref
  484       * datum.  Generally speaking you should call trans_to_datum()
  485       * instead, but sometimes we call trans_lvalue() directly as a
  486       * means of asserting that a particular expression is an lvalue. */
  487  
  488      return match bcx.tcx().adjustments.find(&expr.id) {
  489          None => trans_lvalue_unadjusted(bcx, expr),
  490          Some(_) => {
  491              bcx.sess().span_bug(
  492                  expr.span,
  493                  fmt!("trans_lvalue() called on an expression \
  494                        with adjustments"));
  495          }
  496      };
  497  }

  499  fn trans_to_datum_unadjusted(bcx: @mut Block, expr: @ast::Expr) -> DatumBlock {
  500      /*!
  501       * Translates an expression into a datum.  If this expression
  502       * is an rvalue, this will result in a temporary value being
  503       * created.  If you plan to store the value somewhere else,
  504       * you should prefer `trans_into()` instead.
  505       */
  506  
  507      let mut bcx = bcx;
  508  
  509      debug!("trans_to_datum_unadjusted(expr=%s)", bcx.expr_to_str(expr));
  510      let _indenter = indenter();
  511  
  512      debuginfo::set_source_location(bcx.fcx, expr.id, expr.span);
  513  
  514      match ty::expr_kind(bcx.tcx(), bcx.ccx().maps.method_map, expr) {
  515          ty::LvalueExpr => {
  516              return trans_lvalue_unadjusted(bcx, expr);
  517          }
  518  
  519          ty::RvalueDatumExpr => {
  520              let datum = unpack_datum!(bcx, {
  521                  trans_rvalue_datum_unadjusted(bcx, expr)
  522              });
  523              datum.add_clean(bcx);
  524              return DatumBlock {bcx: bcx, datum: datum};
  525          }
  526  
  527          ty::RvalueStmtExpr => {
  528              bcx = trans_rvalue_stmt_unadjusted(bcx, expr);
  529              return nil(bcx, expr_ty(bcx, expr));
  530          }
  531  
  532          ty::RvalueDpsExpr => {
  533              let ty = expr_ty(bcx, expr);
  534              if ty::type_is_voidish(ty) {
  535                  bcx = trans_rvalue_dps_unadjusted(bcx, expr, Ignore);
  536                  return nil(bcx, ty);
  537              } else {
  538                  let scratch = scratch_datum(bcx, ty, "", false);
  539                  bcx = trans_rvalue_dps_unadjusted(
  540                      bcx, expr, SaveIn(scratch.val));
  541  
  542                  // Note: this is not obviously a good idea.  It causes
  543                  // immediate values to be loaded immediately after a
  544                  // return from a call or other similar expression,
  545                  // which in turn leads to alloca's having shorter
  546                  // lifetimes and hence larger stack frames.  However,
  547                  // in turn it can lead to more register pressure.
  548                  // Still, in practice it seems to increase
  549                  // performance, since we have fewer problems with
  550                  // morestack churn.
  551                  let scratch = scratch.to_appropriate_datum(bcx);
  552  
  553                  scratch.add_clean(bcx);
  554                  return DatumBlock {bcx: bcx, datum: scratch};
  555              }
  556          }
  557      }
