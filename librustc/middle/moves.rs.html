  11  /*!
  12  
  13  # Moves Computation
  14  
  15  The goal of this file is to compute which
  16  expressions/patterns/captures correspond to *moves*.  This is
  17  generally a function of the context in which the expression appears as
  18  well as the expression's type.
  19  
  20  ## Examples
  21  
  22  We will use the following fragment of code to explain the various
  23  considerations.  Note that in this code `x` is used after it has been
  24  moved here.  This is not relevant to this pass, though the information
  25  we compute would later be used to detect this error (see the section
  26  Enforcement of Moves, below).
  27  
  28      struct Foo { a: int, b: ~int }
  29      let x: Foo = ...;
  30      let w = (x {Read}).a;      // Read
  31      let y = (x {Move}).b;      // Move
  32      let z = copy (x {Read}).b; // Read
  33  
  34  Let's look at these examples one by one.  In the first case, `w`, the
  35  expression being assigned is `x.a`, which has `int` type.  In that
  36  case, the value is read, and the container (`x`) is also read.
  37  
  38  In the second case, `y`, `x.b` is being assigned which has type
  39  `~int`.  Because this type moves by default, that will be a move
  40  reference.  Whenever we move from a compound expression like `x.b` (or
  41  `x[b]` or `*x` or `{x)[b].c`, etc), this invalidates all containing
  42  expressions since we do not currently permit "incomplete" variables
  43  where part of them has been moved and part has not.  In this case,
  44  this means that the reference to `x` is also a move.  We'll see later,
  45  though, that these kind of "partial moves", where part of the
  46  expression has been moved, are classified and stored somewhat
  47  differently.
  48  
  49  The final example (`z`) is `copy x.b`: in this case, although the
  50  expression being assigned has type `~int`, there are no moves
  51  involved.
  52  
  53  ### Patterns
  54  
  55  For each binding in a match or let pattern, we also compute a read
  56  or move designation.  A move binding means that the value will be
  57  moved from the value being matched.  As a result, the expression
  58  being matched (aka, the 'discriminant') is either moved or read
  59  depending on whethe the bindings move the value they bind to out of
  60  the discriminant.
  61  
  62  For examples, consider this match expression:
  63  
  64      match x {Move} {
  65        Foo { a: a {Read}, b: b {Move} } => {...}
  66      }
  67  
  68  Here, the binding `b` is value (not ref) mode, and `b` has type
  69  `~int`, and therefore the discriminant expression `x` would be
  70  incomplete so it also considered moved.
  71  
  72  In the following two examples, in contrast, the mode of `b` is either
  73  `copy` or `ref` and hence the overall result is a read:
  74  
  75      match x {Read} {
  76        Foo { a: a {Read}, b: copy b {Read} } => {...}
  77      }
  78  
  79      match x {Read} {
  80        Foo { a: a {Read}, b: ref b {Read} } => {...}
  81      }
  82  
  83  Similar reasoning can be applied to `let` expressions:
  84  
  85      let Foo { a: a {Read}, b: b {Move} } = x {Move};
  86      let Foo { a: a {Read}, b: copy b {Read} } = x {Read};
  87      let Foo { a: a {Read}, b: ref b  {Read} } = x {Read};
  88  
  89  ## Output
  90  
  91  The pass results in the struct `MoveMaps` which contains several
  92  maps:
  93  
  94  `moves_map` is a set containing the id of every *outermost expression* or
  95  *binding* that causes a move.  Note that `moves_map` only contains the *outermost
  96  expressions* that are moved.  Therefore, if you have a use of `x.b`,
  97  as in the example `y` above, the expression `x.b` would be in the
  98  `moves_map` but not `x`.  The reason for this is that, for most
  99  purposes, it's only the outermost expression that is needed.  The
 100  borrow checker and trans, for example, only care about the outermost
 101  expressions that are moved.  It is more efficient therefore just to
 102  store those entries.
 103  
 104  Sometimes though we want to know the variables that are moved (in
 105  particular in the borrow checker). For these cases, the set
 106  `moved_variables_set` just collects the ids of variables that are
 107  moved.
 108  
 109  Finally, the `capture_map` maps from the node_id of a closure
 110  expression to an array of `CaptureVar` structs detailing which
 111  variables are captured and how (by ref, by copy, by move).
 112  
 113  ## Enforcement of Moves
 114  
 115  The enforcement of moves is done by the borrow checker.  Please see
 116  the section "Moves and initialization" in `middle/borrowck/doc.rs`.
 117  
 118  ## Distributive property
 119  
 120  Copies are "distributive" over parenthesization, but blocks are
 121  considered rvalues.  What this means is that, for example, neither
 122  `a.clone()` nor `(a).clone()` will move `a` (presuming that `a` has a
 123  linear type and `clone()` takes its self by reference), but
 124  `{a}.clone()` will move `a`, as would `(if cond {a} else {b}).clone()`
 125  and so on.
 126  
 127  */

 181  struct VisitContext {

 183      method_map: method_map,
 184      move_maps: MoveMaps
 185  }

 187  #[deriving(Eq)]
 188  enum UseMode {
 189      Move,        // This value or something owned by it is moved.
 190      Read         // Read no matter what the type.
 191  }

 193  pub fn compute_moves(tcx: ty::ctxt,
 194                       method_map: method_map,
 195                       crate: &Crate) -> MoveMaps
 196  {
 197      let visitor = oldvisit::mk_vt(@oldvisit::Visitor {
 198          visit_fn: compute_modes_for_fn,
 199          visit_expr: compute_modes_for_expr,
 200          visit_local: compute_modes_for_local,
 201          .. *oldvisit::default_visitor()
 202      });
 203      let visit_cx = VisitContext {
 204          tcx: tcx,
 205          method_map: method_map,
 206          move_maps: MoveMaps {
 207              moves_map: @mut HashSet::new(),
 208              capture_map: @mut HashMap::new(),
 209              moved_variables_set: @mut HashSet::new()
 210          }
 211      };
 212      oldvisit::visit_crate(crate, (visit_cx, visitor));
 213      return visit_cx.move_maps;
 214  }

 216  pub fn moved_variable_node_id_from_def(def: def) -> Option<NodeId> {
 217      match def {
 218        def_binding(nid, _) |
 219        def_arg(nid, _) |
 220        def_local(nid, _) |
 221        def_self(nid, _) => Some(nid),
 222  
 223        _ => None
 224      }
 225  }

 230  fn compute_modes_for_local<'a>(local: @Local,
 231                                 (cx, v): (VisitContext,
 232                                           vt<VisitContext>)) {
 233      cx.use_pat(local.pat);
 234      for &init in local.init.iter() {
 235          cx.use_expr(init, Read, v);
 236      }
 237  }

 239  fn compute_modes_for_fn(fk: &oldvisit::fn_kind,
 240                          decl: &fn_decl,
 241                          body: &Block,
 242                          span: span,
 243                          id: NodeId,
 244                          (cx, v): (VisitContext,
 245                                    vt<VisitContext>)) {
 246      for a in decl.inputs.iter() {
 247          cx.use_pat(a.pat);
 248      }
 249      oldvisit::visit_fn(fk, decl, body, span, id, (cx, v));
 250  }

 259  impl VisitContext {
 260      pub fn consume_exprs(&self, exprs: &[@expr], visitor: vt<VisitContext>) {
 261          for expr in exprs.iter() {
 262              self.consume_expr(*expr, visitor);
 263          }
 264      }
 265  
 266      pub fn consume_expr(&self, expr: @expr, visitor: vt<VisitContext>) {
 267          /*!
 268           * Indicates that the value of `expr` will be consumed,
 269           * meaning either copied or moved depending on its type.
 270           */
 271  
 272          debug!("consume_expr(expr=%s)",
 273                 expr.repr(self.tcx));
 274  
 275          let expr_ty = ty::expr_ty_adjusted(self.tcx, expr);
 276          if ty::type_moves_by_default(self.tcx, expr_ty) {
 277              self.move_maps.moves_map.insert(expr.id);
 278              self.use_expr(expr, Move, visitor);
 279          } else {
 280              self.use_expr(expr, Read, visitor);
 281          };
 282      }
 283  
 284      pub fn consume_block(&self, blk: &Block, visitor: vt<VisitContext>) {
 285          /*!
 286           * Indicates that the value of `blk` will be consumed,
 287           * meaning either copied or moved depending on its type.
 288           */
 289  
 290          debug!("consume_block(blk.id=%?)", blk.id);
 291  
 292          for stmt in blk.stmts.iter() {
 293              (visitor.visit_stmt)(*stmt, (*self, visitor));
 294          }
 295  
 296          for tail_expr in blk.expr.iter() {
 297              self.consume_expr(*tail_expr, visitor);
 298          }
 299      }

 301      pub fn use_expr(&self,
 302                      expr: @expr,
 303                      expr_mode: UseMode,
 304                      visitor: vt<VisitContext>) {
 305          /*!
 306           * Indicates that `expr` is used with a given mode.  This will
 307           * in turn trigger calls to the subcomponents of `expr`.
 308           */
 309  
 310          debug!("use_expr(expr=%s, mode=%?)",
 311                 expr.repr(self.tcx),
 312                 expr_mode);
 313  
 314          // `expr_mode` refers to the post-adjustment value.  If one of
 315          // those adjustments is to take a reference, then it's only
 316          // reading the underlying expression, not moving it.
 317          let comp_mode = match self.tcx.adjustments.find(&expr.id) {
 318              Some(&@ty::AutoDerefRef(
 319                  ty::AutoDerefRef {
 320                      autoref: Some(_), _})) => Read,
 321              _ => expr_mode
 322          };
 323  
 324          debug!("comp_mode = %?", comp_mode);
 325  
 326          match expr.node {
 327              expr_path(*) | expr_self => {
 328                  match comp_mode {
 329                      Move => {
 330                          let def = self.tcx.def_map.get_copy(&expr.id);
 331                          let r = moved_variable_node_id_from_def(def);
 332                          for &id in r.iter() {
 333                              self.move_maps.moved_variables_set.insert(id);
 334                          }
 335                      }
 336                      Read => {}
 337                  }
 338              }
 339  
 340              expr_unary(_, deref, base) => {       // *base
 341                  if !self.use_overloaded_operator(
 342                      expr, base, [], visitor)
 343                  {
 344                      // Moving out of *base moves out of base.
 345                      self.use_expr(base, comp_mode, visitor);
 346                  }
 347              }
 348  
 349              expr_field(base, _, _) => {        // base.f
 350                  // Moving out of base.f moves out of base.
 351                  self.use_expr(base, comp_mode, visitor);
 352              }
 353  
 354              expr_index(_, lhs, rhs) => {          // lhs[rhs]
 355                  if !self.use_overloaded_operator(
 356                      expr, lhs, [rhs], visitor)
 357                  {
 358                      self.use_expr(lhs, comp_mode, visitor);
 359                      self.consume_expr(rhs, visitor);
 360                  }
 361              }
 362  
 363              expr_call(callee, ref args, _) => {    // callee(args)
 364                  // Figure out whether the called function is consumed.
 365                  let mode = match ty::get(ty::expr_ty(self.tcx, callee)).sty {
 366                      ty::ty_closure(ref cty) => {
 367                          match cty.onceness {
 368                          Once => Move,
 369                          Many => Read,
 370                          }
 371                      },
 372                      ty::ty_bare_fn(*) => Read,
 373                      ref x =>
 374                          self.tcx.sess.span_bug(callee.span,
 375                              fmt!("non-function type in moves for expr_call: %?", x)),
 376                  };
 377                  // Note we're not using consume_expr, which uses type_moves_by_default
 378                  // to determine the mode, for this. The reason is that while stack
 379                  // closures should be noncopyable, they shouldn't move by default;
 380                  // calling a closure should only consume it if it's once.
 381                  if mode == Move {
 382                      self.move_maps.moves_map.insert(callee.id);
 383                  }
 384                  self.use_expr(callee, mode, visitor);
 385                  self.use_fn_args(callee.id, *args, visitor);
 386              }
 387  
 388              expr_method_call(callee_id, rcvr, _, _, ref args, _) => { // callee.m(args)
 389                  // Implicit self is equivalent to & mode, but every
 390                  // other kind should be + mode.
 391                  self.use_receiver(rcvr, visitor);
 392                  self.use_fn_args(callee_id, *args, visitor);
 393              }
 394  
 395              expr_struct(_, ref fields, opt_with) => {
 396                  for field in fields.iter() {
 397                      self.consume_expr(field.expr, visitor);
 398                  }
 399  
 400                  for with_expr in opt_with.iter() {
 401                      // If there are any fields whose type is move-by-default,
 402                      // then `with` is consumed, otherwise it is only read
 403                      let with_ty = ty::expr_ty(self.tcx, *with_expr);
 404                      let with_fields = match ty::get(with_ty).sty {
 405                          ty::ty_struct(did, ref substs) => {
 406                              ty::struct_fields(self.tcx, did, substs)
 407                          }
 408                          ref r => {
 409                             self.tcx.sess.span_bug(
 410                                  with_expr.span,
 411                                  fmt!("bad base expr type in record: %?", r))
 412                          }
 413                      };
 414  
 415                      // The `with` expr must be consumed if it contains
 416                      // any fields which (1) were not explicitly
 417                      // specified and (2) have a type that
 418                      // moves-by-default:
 419                      let consume_with = with_fields.iter().any(|tf| {
 420                          !fields.iter().any(|f| f.ident == tf.ident) &&
 421                              ty::type_moves_by_default(self.tcx, tf.mt.ty)
 422                      });
 423  
 424                      if consume_with {
 425                          self.consume_expr(*with_expr, visitor);
 426                      } else {
 427                          self.use_expr(*with_expr, Read, visitor);
 428                      }
 429                  }
 430              }
