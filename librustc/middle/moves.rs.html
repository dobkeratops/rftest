  11  /*!
  12  
  13  # Moves Computation
  14  
  15  The goal of this file is to compute which
  16  expressions/patterns/captures correspond to *moves*.  This is
  17  generally a function of the context in which the expression appears as
  18  well as the expression's type.
  19  
  20  ## Examples
  21  
  22  We will use the following fragment of code to explain the various
  23  considerations.  Note that in this code `x` is used after it has been
  24  moved here.  This is not relevant to this pass, though the information
  25  we compute would later be used to detect this error (see the section
  26  Enforcement of Moves, below).
  27  
  28      struct Foo { a: int, b: ~int }
  29      let x: Foo = ...;
  30      let w = (x {Read}).a;      // Read
  31      let y = (x {Move}).b;      // Move
  32      let z = copy (x {Read}).b; // Read
  33  
  34  Let's look at these examples one by one.  In the first case, `w`, the
  35  expression being assigned is `x.a`, which has `int` type.  In that
  36  case, the value is read, and the container (`x`) is also read.
  37  
  38  In the second case, `y`, `x.b` is being assigned which has type
  39  `~int`.  Because this type moves by default, that will be a move
  40  reference.  Whenever we move from a compound expression like `x.b` (or
  41  `x[b]` or `*x` or `{x}[b].c`, etc), this invalidates all containing
  42  expressions since we do not currently permit "incomplete" variables
  43  where part of them has been moved and part has not.  In this case,
  44  this means that the reference to `x` is also a move.  We'll see later,
  45  though, that these kind of "partial moves", where part of the
  46  expression has been moved, are classified and stored somewhat
  47  differently.
  48  
  49  The final example (`z`) is `copy x.b`: in this case, although the
  50  expression being assigned has type `~int`, there are no moves
  51  involved.
  52  
  53  ### Patterns
  54  
  55  For each binding in a match or let pattern, we also compute a read
  56  or move designation.  A move binding means that the value will be
  57  moved from the value being matched.  As a result, the expression
  58  being matched (aka, the 'discriminant') is either moved or read
  59  depending on whether the bindings move the value they bind to out of
  60  the discriminant.
  61  
  62  For examples, consider this match expression:
  63  
  64      match x {Move} {
  65        Foo { a: a {Read}, b: b {Move} } => {...}
  66      }
  67  
  68  Here, the binding `b` is value (not ref) mode, and `b` has type
  69  `~int`, and therefore the discriminant expression `x` would be
  70  incomplete so it also considered moved.
  71  
  72  In the following two examples, in contrast, the mode of `b` is either
  73  `copy` or `ref` and hence the overall result is a read:
  74  
  75      match x {Read} {
  76        Foo { a: a {Read}, b: copy b {Read} } => {...}
  77      }
  78  
  79      match x {Read} {
  80        Foo { a: a {Read}, b: ref b {Read} } => {...}
  81      }
  82  
  83  Similar reasoning can be applied to `let` expressions:
  84  
  85      let Foo { a: a {Read}, b: b {Move} } = x {Move};
  86      let Foo { a: a {Read}, b: copy b {Read} } = x {Read};
  87      let Foo { a: a {Read}, b: ref b  {Read} } = x {Read};
  88  
  89  ## Output
  90  
  91  The pass results in the struct `MoveMaps` which contains several
  92  maps:
  93  
  94  `moves_map` is a set containing the id of every *outermost expression* or
  95  *binding* that causes a move.  Note that `moves_map` only contains the *outermost
  96  expressions* that are moved.  Therefore, if you have a use of `x.b`,
  97  as in the example `y` above, the expression `x.b` would be in the
  98  `moves_map` but not `x`.  The reason for this is that, for most
  99  purposes, it's only the outermost expression that is needed.  The
 100  borrow checker and trans, for example, only care about the outermost
 101  expressions that are moved.  It is more efficient therefore just to
 102  store those entries.
 103  
 104  Sometimes though we want to know the variables that are moved (in
 105  particular in the borrow checker). For these cases, the set
 106  `moved_variables_set` just collects the ids of variables that are
 107  moved.
 108  
 109  Finally, the `capture_map` maps from the node_id of a closure
 110  expression to an array of `CaptureVar` structs detailing which
 111  variables are captured and how (by ref, by copy, by move).
 112  
 113  ## Enforcement of Moves
 114  
 115  The enforcement of moves is done by the borrow checker.  Please see
 116  the section "Moves and initialization" in `middle/borrowck/doc.rs`.
 117  
 118  ## Distributive property
 119  
 120  Copies are "distributive" over parenthesization, but blocks are
 121  considered rvalues.  What this means is that, for example, neither
 122  `a.clone()` nor `(a).clone()` will move `a` (presuming that `a` has a
 123  linear type and `clone()` takes its self by reference), but
 124  `{a}.clone()` will move `a`, as would `(if cond {a} else {b}).clone()`
 125  and so on.
 126  
 127  */

 246                          body: &Block,

 268      pub fn consume_expr(&mut self, expr: &Expr) {
 269          /*!
 270           * Indicates that the value of `expr` will be consumed,
 271           * meaning either copied or moved depending on its type.
 272           */
 273  
 274          debug!("consume_expr(expr={})",
 275                 expr.repr(self.tcx));
 276  
 277          let expr_ty = ty::expr_ty_adjusted(self.tcx, expr,
 278                                             &*self.method_map.borrow());
 279          if ty::type_moves_by_default(self.tcx, expr_ty) {
 280              self.move_maps.moves_map.insert(expr.id);
 281              self.use_expr(expr, Move);
 282          } else {
 283              self.use_expr(expr, Read);
 284          };
 285      }
 286  
 287      pub fn consume_block(&mut self, blk: &Block) {
 288          /*!
 289           * Indicates that the value of `blk` will be consumed,
 290           * meaning either copied or moved depending on its type.
 291           */
 292  
 293          debug!("consume_block(blk.id={:?})", blk.id);
 294  
 295          for stmt in blk.stmts.iter() {
 296              self.visit_stmt(*stmt, ());
 297          }
 298  
 299          for tail_expr in blk.expr.iter() {
 300              self.consume_expr(*tail_expr);
 301          }
 302      }

 304      pub fn use_expr(&mut self,
 305                      expr: &Expr,
 306                      expr_mode: UseMode) {
 307          /*!
 308           * Indicates that `expr` is used with a given mode.  This will
 309           * in turn trigger calls to the subcomponents of `expr`.
 310           */
 311  
 312          debug!("use_expr(expr={}, mode={:?})",
 313                 expr.repr(self.tcx),
 314                 expr_mode);
 315  
 316          // `expr_mode` refers to the post-adjustment value.  If one of
 317          // those adjustments is to take a reference, then it's only
 318          // reading the underlying expression, not moving it.
 319          let comp_mode = match self.tcx.adjustments.borrow().find(&expr.id) {
 320              Some(adjustment) => {
 321                  match **adjustment {
 322                      ty::AutoDerefRef(ty::AutoDerefRef {
 323                          autoref: Some(_),
 324                          ..
 325                      }) => Read,
 326                      _ => expr_mode,
 327                  }
 328              }
 329              _ => expr_mode,
 330          };
 331  
 332          debug!("comp_mode = {:?}", comp_mode);
 333  
 334          match expr.node {
 335              ExprPath(..) => {
 336                  match comp_mode {
 337                      Move => {
 338                          let def = self.tcx.def_map.borrow().get_copy(&expr.id);
 339                          let r = moved_variable_node_id_from_def(def);
 340                          for &id in r.iter() {
 341                              self.move_maps.moved_variables_set.insert(id);
 342                          }
 343                      }
 344                      Read => {}
 345                  }
 346              }
 347  
 348              ExprUnary(UnDeref, base) => {      // *base
 349                  if !self.use_overloaded_operator(expr, base, []) {
 350                      // Moving out of *base moves out of base.
 351                      self.use_expr(base, comp_mode);
 352                  }
 353              }
 354  
 355              ExprField(base, _, _) => {         // base.f
 356                  // Moving out of base.f moves out of base.
 357                  self.use_expr(base, comp_mode);
 358              }
 359  
 360              ExprIndex(lhs, rhs) => {           // lhs[rhs]
 361                  if !self.use_overloaded_operator(expr, lhs, [rhs]) {
 362                      self.use_expr(lhs, comp_mode);
 363                      self.consume_expr(rhs);
 364                  }
 365              }
 366  
 367              ExprCall(callee, ref args) => {    // callee(args)
 368                  // Figure out whether the called function is consumed.
 369                  let mode = match ty::get(ty::expr_ty(self.tcx, callee)).sty {
 370                      ty::ty_closure(ref cty) => {
 371                          match cty.onceness {
 372                          Once => Move,
 373                          Many => Read,
 374                          }
 375                      },
 376                      ty::ty_bare_fn(..) => Read,
 377                      ref x =>
 378                          self.tcx.sess.span_bug(callee.span,
 379                              format!("non-function type in moves for expr_call: {:?}", x)),
 380                  };
 381                  // Note we're not using consume_expr, which uses type_moves_by_default
 382                  // to determine the mode, for this. The reason is that while stack
 383                  // closures should be noncopyable, they shouldn't move by default;
 384                  // calling a closure should only consume it if it's once.
 385                  if mode == Move {
 386                      self.move_maps.moves_map.insert(callee.id);
 387                  }
 388                  self.use_expr(callee, mode);
 389                  self.use_fn_args(args.as_slice());
 390              }
 391  
 392              ExprMethodCall(_, _, ref args) => { // callee.m(args)
 393                  self.use_fn_args(args.as_slice());
 394              }
 395  
 396              ExprStruct(_, ref fields, opt_with) => {
 397                  for field in fields.iter() {
 398                      self.consume_expr(field.expr);
 399                  }
 400  
 401                  for with_expr in opt_with.iter() {
 402                      // If there are any fields whose type is move-by-default,
 403                      // then `with` is consumed, otherwise it is only read
 404                      let with_ty = ty::expr_ty(self.tcx, *with_expr);
 405                      let with_fields = match ty::get(with_ty).sty {
 406                          ty::ty_struct(did, ref substs) => {
 407                              ty::struct_fields(self.tcx, did, substs)
 408                          }
 409                          ref r => {
 410                             self.tcx.sess.span_bug(
 411                                  with_expr.span,
 412                                  format!("bad base expr type in record: {:?}", r))
 413                          }
 414                      };
 415  
 416                      // The `with` expr must be consumed if it contains
 417                      // any fields which (1) were not explicitly
 418                      // specified and (2) have a type that
 419                      // moves-by-default:
 420                      let consume_with = with_fields.iter().any(|tf| {
 421                          !fields.iter().any(|f| f.ident.node.name == tf.ident.name) &&
 422                              ty::type_moves_by_default(self.tcx, tf.mt.ty)
 423                      });

 425                      fn has_dtor(tcx: &ty::ctxt, ty: ty::t) -> bool {
 426                          use middle::ty::{get,ty_struct,ty_enum};
 427                          match get(ty).sty {
 428                              ty_struct(did, _) | ty_enum(did, _) => ty::has_dtor(tcx, did),
 429                              _ => false,
 430                          }
